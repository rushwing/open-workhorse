# Scenario

一次完整的 request→response 往返，两条消息均进入各自 done/，通过 thread_id 可重建链路。

# Steps

1. 在 `for-menglan/done/` 预置 type=request（thread_id=T，correlation_id=C，req_id=REQ-035）
2. 在 `for-pandas/pending/` 预置 type=response（thread_id=T，correlation_id=C，legacy_type=dev_complete）
3. 调用 `inbox_read_pandas`

# Expected

- response 移到 `for-pandas/done/`
- `for-menglan/done/` 中的 request 文件仍在（未被删除）
- 两个 done/ 目录中的文件均含 `thread_id: T`（即可通过 grep thread_id 重建完整链路）
