# Scenario

`inbox_write_v2()` 在写入消息时，若 payload 的 `context_summary` 字段超过 500 字符，自动截断并发出 warn，写入文件中的值不超过 500 字符。

# Steps

1. 设置 SHARED_RESOURCES_ROOT 为临时目录
2. 创建 payload 文件，`context_summary:` 值为 600 字符字符串
3. 调用 `inbox_write_v2 "menglan" "request" "implement" "thread_036_5" "corr_036_5" "" "P2" "false" "<payload_file>"`
4. 读取生成文件中 `context_summary:` 值的长度

# Expected

- 标准错误包含 "context_summary truncated" 字样
- 生成文件中 `context_summary:` 值长度 ≤ 500 字符
