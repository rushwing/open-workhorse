# Scenario

`inbox_write_v2()` 在写入消息时，若 payload 的 `references` 块包含 `type:` 值不在合法枚举（`req|pr|bug|doc|file`）内，发出 warn 日志（非阻断，消息仍写入）。

# Steps

1. 设置 SHARED_RESOURCES_ROOT 为临时目录
2. 创建 payload 文件，包含 references 块，其中 `type: invalid_type`（枚举外的值）
3. 调用 `inbox_write_v2 "menglan" "request" "implement" "thread_036_6" "corr_036_6" "" "P2" "false" "<payload_file>"`
4. 检查 stderr

# Expected

- 标准错误包含 "references type" 和 "not in enum" 字样
- 消息文件仍然被写入（非阻断）
