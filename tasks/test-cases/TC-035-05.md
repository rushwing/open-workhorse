# Scenario

for-menglan/done/ 中已有一条携带已知 thread_id 的消息，连续调用两次 `thread_get_or_create`。

# Steps

1. 在 `for-menglan/done/` 预置消息，frontmatter 含 `thread_id: thread_REQ-035_9999999`，payload 含 `req_id: REQ-035`
2. 在同一 shell 脚本中连续调用 `thread_get_or_create "REQ-035"` 两次
3. 比较两次输出

# Expected

- 两次输出相同
- 输出值等于 `thread_REQ-035_9999999`
