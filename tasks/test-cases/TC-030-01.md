# Scenario

REQ 从 draft 推进到 done 的完整主路径，经过所有中间状态。

# Steps

1. 构造 req fixture：`status=draft, tc_policy=required`
2. `draft → review_ready`；断言 status=review_ready
3. `review_ready → req_review`；断言 status=req_review, owner=huahua
4. `req_review → ready`；断言 status=ready, owner=huahua
5. `ready → test_designed`；断言 status=test_designed, owner=huahua
6. `test_designed → in_progress`；断言 status=in_progress, owner=claude_code
7. `in_progress → review`；断言 status=review
8. `review → done`；断言 status=done, owner=unassigned

# Expected

- 每步 status/owner 符合 requirement-standard.md §8.4 路由表
- done 时 owner=unassigned
