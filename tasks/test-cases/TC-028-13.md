# Scenario

user_bug 处于 regressing，通知发出仅 5 天，未超过 14 天阈值，同步不触发关闭，剩余天数计算正确。

# Steps

1. 构造 bug fixture：`bug_type=user_bug, status=regressing, github_issue=42, regressing_notified=true, regressing_notified_at=TODAY-5`
2. Mock `gh.issueView(42)` 返回 `{state: "OPEN"}`
3. Mock `gh.issueClose` 函数
4. 运行 GitHub 同步流程
5. 断言 `gh.issueClose` **未被调用**
6. 断言 `bug.status=regressing`（状态未变）
7. 断言同步日志或返回值中剩余天数=9（14 - 5 = 9）

# Expected

- 5 天未达 14 天阈值，不触发自动关闭
- gh.issueClose 未被调用
- status 保持 regressing
- 剩余天数=9 可从日志或返回值中验证
