# Scenario

当 REQ 的 `pending_bugs` 引用一个不存在的 BUG 文件时，`check-req-coverage.sh` 应报错并退出非零。

# Steps

1. 确认 `tasks/bugs/BUG-999.md` 不存在
2. 在 `tasks/features/` 创建临时 REQ 文件，frontmatter 含 `pending_bugs: [BUG-999]`，其他字段合法
3. 执行 `bash scripts/check-req-coverage.sh`
4. 检查退出码和错误输出

# Expected

- 退出码：1（非零）
- 输出含 "BUG-999 不存在" 或等价描述（如 "tasks/bugs/BUG-999.md not found"）
- 明确指向引用来源（REQ 文件名或 req_id）
