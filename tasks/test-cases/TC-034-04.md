# Scenario

`_dispatch_msg()` 返回非零退出码时，消息应移到 `failed/` 并追加错误摘要行。

# Steps

1. 设置 SHARED_RESOURCES_ROOT 为临时目录
2. 在 `for-pandas/pending/` 写入消息
3. Override `_dispatch_msg()` 为 `return 1`（模拟 handler 失败）
4. 调用 `inbox_init; inbox_read_pandas`

# Expected

- `for-pandas/failed/` 下有与原文件同名的文件
- 文件末尾包含 `ERROR:` 行
- `for-pandas/pending/` 下该文件不存在
