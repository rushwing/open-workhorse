# Scenario

`mv pending/$f claimed/$f` 失败，但源文件仍然存在（非竞争场景，如磁盘满、目标目录权限异常）。
应输出错误日志（不静默跳过），且文件留在 `pending/` 中，不被移走或删除。

# Steps

1. 设置 SHARED_RESOURCES_ROOT 为临时目录
2. 在 `for-pandas/pending/` 写入消息文件
3. Override bash `mv` 函数：当参数为 `pending/ -> claimed/` 时，不删除源文件，直接返回 1
   （模拟真实文件系统错误，源文件完好）
4. 调用 `inbox_init; inbox_read_pandas`

# Expected

- 退出码为 0（其他消息不受影响）
- stderr 包含错误日志（`[pandas]` 前缀 + "Claim mv" 关键字）
- 消息文件仍在 `pending/`（未被静默消费）
