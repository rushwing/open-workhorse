# Scenario

同一 REQ 有两条 request 轨迹（不同 correlation_id），response 携带第二条 request 的
correlation_id，应通过校验。验证 _corr_valid_for_req 检查所有 request，而非仅第一条。

# Steps

1. 在 `for-menglan/done/` 预置 request-1（correlation_id=C1，req_id=REQ-035）
2. 在 `for-menglan/done/` 预置 request-2（correlation_id=C2，req_id=REQ-035）
3. 在 `for-pandas/pending/` 写入 type=response（correlation_id=C2，legacy_type=dev_complete）
4. 调用 `inbox_read_pandas`

# Expected

- response 移到 `for-pandas/done/`（C2 在轨迹中找得到，校验通过）
- 退出码为 0
- 无 warn/error 含 correlation_id 字样
