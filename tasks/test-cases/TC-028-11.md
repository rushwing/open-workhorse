# Scenario

user_bug 进入 regressing 时发送验收通知。同步运行两次，第一次发通知并写 regressing_notified=true；第二次不重复发通知。

# Steps

1. 构造 bug fixture：`bug_type=user_bug, status=regressing, github_issue=42, regressing_notified=false`
2. Mock `gh.issueComment` 函数；mock 当前日期为 TODAY
3. 第一次运行同步
4. 断言 `gh.issueComment` 被调用恰好 1 次
5. 断言 `bug.regressing_notified=true`
6. 断言 `bug.regressing_notified_at=TODAY`（ISO 日期格式）
7. 第二次运行同步（状态不变，issue 仍 OPEN）
8. 断言 `gh.issueComment` 总调用次数仍为 1（未再次调用）

# Expected

- 验收通知仅发送一次
- regressing_notified=true 写入后，后续同步跳过通知逻辑
- regressing_notified_at 写入 TODAY
