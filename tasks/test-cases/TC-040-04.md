# Scenario

某 fixture branch（如 feat/REQ-E002）在 mock 环境中不存在，
脚本应 gracefully 跳过该 fixture，标记为 "not_run"，不计入 pass/fail，不崩溃。

# Steps

1. mock：仅 REQ-E001 的 git log 存在且合规；REQ-E002 和 REQ-E003 的 `git log` 返回空或 non-zero exit
2. 调用 eval:compliance
3. 检查退出码和 last-run.md

# Expected

- 脚本不崩溃（无 unhandled exception / set -e 非预期退出）
- REQ-E002 和 REQ-E003 在报告中标记为 `not_run` 或 `-`，不显示 ✗
- compliance 仅基于 REQ-E001 的 3 步计算（3/3 = 100%）
- 退出码为 0（仅存在的 branch 全部通过且无回归）
