# Scenario

覆盖两个子场景：

**Sub-case A（部分缺失）：** 某些 fixture branch 不存在，脚本应 gracefully 跳过，
标记为 `not_run`，不计入 pass/fail，不崩溃。

**Sub-case B（全部缺失）：** 所有 fixture branch 均不存在，脚本应 exit 1 并输出
"no eval sessions found" 错误信息。

# Steps

## Sub-case A

1. mock：仅 REQ-E001 的 git log 存在且合规；REQ-E002 和 REQ-E003 的 `git log` 返回空或 non-zero exit
2. 调用 eval:compliance
3. 检查退出码和 last-run.md

## Sub-case B

1. mock：REQ-E001、REQ-E002、REQ-E003 的 `git log` 全部返回空或 non-zero exit
2. 调用 eval:compliance
3. 检查退出码和错误输出

# Expected

## Sub-case A

- 脚本不崩溃（无 unhandled exception / set -e 非预期退出）
- REQ-E002 和 REQ-E003 在报告中标记为 `not_run` 或 `-`，不显示 ✗
- compliance 仅基于 REQ-E001 的 3 步计算（3/3 = 100%）
- 退出码为 0（仅存在的 branch 全部通过且无回归）

## Sub-case B

- 退出码为 1
- stderr 或报告中包含 "no eval sessions found" 字样
- 脚本不崩溃（no unhandled exception）
