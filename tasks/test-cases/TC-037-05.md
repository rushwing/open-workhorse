# Scenario

Menglan が `~/workspace-menglan/open-workhorse/` worktree 内でファイルを変更・ステージ
しても、主 checkout（`~/workspace-pandas/open-workhorse/`）の git status が
クリーンなままであることを確認する。

# Steps

1. `./scripts/harness.sh implement REQ-037` で worktree を作成
2. `echo "scratch" > ~/workspace-menglan/open-workhorse/SCRATCH_TEST.tmp` でファイル作成
3. `git -C ~/workspace-menglan/open-workhorse add SCRATCH_TEST.tmp` でステージ
4. `git -C ~/workspace-menglan/open-workhorse status` で worktree 側の変更を確認
5. `git status` を主 checkout ディレクトリ（`~/workspace-pandas/open-workhorse/`）から実行
6. 後片付け：`git -C ~/workspace-menglan/open-workhorse restore --staged SCRATCH_TEST.tmp && rm ~/workspace-menglan/open-workhorse/SCRATCH_TEST.tmp`

# Expected

- Step 4：worktree 内では `SCRATCH_TEST.tmp` が staged として表示される
- Step 5：主 checkout の `git status` は `nothing to commit, working tree clean` であること
- 主 checkout のインデックスが worktree の変更に影響されないこと
