title: "_check_stall_and_keepalive dedup — no duplicate when keep-alive already pending"
layer: L2
status: pass

When a stale REQ already has a keep-alive message in `inbox/{owner}/pending/` or `claimed/`, `_check_stall_and_keepalive` must skip sending a second one to prevent flooding.
