# Scenario

`inbox_write_v2()` 写入 type=request 消息时，payload 缺少 `done_criteria` 字段，预期 warn 日志 + envelope 中写入 `delegation_incomplete: true`，但消息仍然写入文件（非阻断）。

# Steps

1. 设置 SHARED_RESOURCES_ROOT 为临时目录
2. 创建 payload 文件，包含 `objective:`、`scope:`、`expected_output:`，但不含 `done_criteria:`
3. 调用 `inbox_write_v2 "menglan" "request" "implement" "thread_036_2" "corr_036_2" "" "P1" "true" "<payload_file>"`
4. 检查 stderr 和生成文件内容

# Expected

- 标准错误包含 "delegation incomplete" 字样，并提到 `done_criteria`
- 生成文件包含 `delegation_incomplete: true`
- 文件仍然被写入（不因校验失败而中止）
