# Scenario

`inbox_write()` 作为 @deprecated wrapper 仍可调用，内部委托给 `inbox_write_v2()`，
调用方无需改动，生成文件包含 ATM Envelope 字段及旧 legacy 字段（req_id, summary 等）。

# Steps

1. 调用 `inbox_write "menglan" "implement" "REQ-033" "实现测试" "" "success" "" ""`
2. 读取 `for-menglan/` 生成的文件内容
3. 验证文件格式

# Expected

- 调用不报错（exit code 0）
- 生成文件包含 `type: request`（新 Envelope）
- 生成文件包含 `action: implement`
- 生成文件包含 `req_id: REQ-033`（legacy payload 字段）
- 生成文件包含 `message_id:` 字段
