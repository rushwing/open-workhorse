# Scenario

`pending/` 目录为空（文件已被其他进程 claim 到 `claimed/`），`inbox_read_pandas` 应静默跳过，不报错。

# Steps

1. 设置 SHARED_RESOURCES_ROOT 为临时目录
2. 将消息文件直接放在 `for-pandas/claimed/`（模拟已被其他进程 claim）
3. `for-pandas/pending/` 为空
4. 调用 `inbox_init; inbox_read_pandas`

# Expected

- 退出码为 0
- stderr 不包含 "ERROR:"
- 没有异常终止
