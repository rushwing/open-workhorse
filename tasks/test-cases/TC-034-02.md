# Scenario

两个并发 pandas 心跳同时扫描 `pending/`，其中一个先完成 `mv pending/$f claimed/$f`，
另一个再执行时源文件已不存在（ENOENT），应静默跳过，不产生错误。

# Steps

1. 设置 SHARED_RESOURCES_ROOT 为临时目录
2. 在 `for-pandas/pending/` 写入消息文件
3. Override bash `mv` 函数：当参数为 `pending/ -> claimed/` 时，删除源文件后返回 1
   （模拟另一个 worker 抢先完成 claim，使源文件在我们的 mv 执行前消失）
4. 调用 `inbox_init; inbox_read_pandas`

# Expected

- 退出码为 0
- stderr 不包含 `ERROR:`（ENOENT 竞争属于正常情况，不应报错）
- 消息文件已从 `pending/` 消失（被模拟竞争者删除）
