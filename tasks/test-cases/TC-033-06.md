# Scenario

`inbox_read_pandas()` 处理 ATM Envelope 格式（type=response）消息时，
通过 `legacy_type: tc_complete` 字段路由到 `_handle_tc_complete`，
并正确处理 ATM canonical status `completed`。

**方向说明**：`implement` 是 Pandas→Menglan 的 request action；
Menglan→Pandas 的完成信号必须是 `type=response`（不是 type=request）。

# Steps

1. 在 `for-pandas/` 写入正确方向的 ATM response 消息：
   ```yaml
   ---
   message_id: msg_test_006
   type: response
   from: menglan
   to: pandas
   ...
   ---
   req_id: REQ-033
   status: completed
   legacy_type: tc_complete
   ```
2. 调用 `inbox_read_pandas`

# Expected

- exit code 0
- `_handle_tc_complete` 被调用（status=completed → success path）
- `for-menglan/` 中产生新的 implement 消息
- 消费后消息文件被删除
