# Scenario

`check-req-coverage.sh` 的 STATUS_ENUM 不包含 `review_wip`，该值应被拒绝以验证枚举校验有效性。

# Steps

1. 在 `tasks/features/` 创建临时 REQ 文件（如 `REQ-TEST-029-02.md`），frontmatter 含 `status: review_wip`，其他必填字段合法
2. 执行 `bash scripts/check-req-coverage.sh`
3. 检查退出码和错误输出

# Expected

- 退出码：1（非零）
- 输出含 "值 'review_wip' 不在允许枚举" 或等价错误描述
- 明确指向该 REQ 文件名或 req_id
