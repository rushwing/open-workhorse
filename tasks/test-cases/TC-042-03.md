# Scenario

`inboxClaim()` 将 `pending/` 中最旧的消息原子移动到 `claimed/`，
并发调用时文件级互斥保证只有一个调用成功 claim。

# Steps

1. 使用 tmp 目录，调用 `inboxInit` 初始化目录结构
2. 写入一条消息到 `pending/`（via `inboxWrite`）
3. 并发调用两次 `inboxClaim(sharedResourcesRoot, "menglan")`（Promise.all）
4. 检查结果

# Expected

- 两次并发调用中，恰好有一次返回 `{ path, envelope }` 对象（非 null）
- 另一次返回 `null`（无消息可 claim，文件已被另一次 claim）
- 成功返回的 `envelope` 字段与写入时一致
- `claimed/` 目录下存在对应文件
- `pending/` 目录下该文件不再存在
