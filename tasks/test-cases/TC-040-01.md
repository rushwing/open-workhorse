# Scenario

`eval-compliance` 脚本分析 3 个全部合规的 mock git log fixtures（REQ-E001/E002/E003），
全部 9 步均通过，应 exit 0 并输出 compliance=100%。

# Steps

1. 在测试中 mock `git log` 输出：3 个 branch 均包含 claim commit（owner 变更 + status→in_progress）、≥1 后续非 claim commit、HEAD REQ 文件 status=review
2. 调用 `npm run eval:compliance`（或等价的 compliance.ts 函数）
3. 检查退出码和 `eval/last-run.md` 内容

# Expected

- 退出码为 0
- `eval/last-run.md` 中每个 fixture 的 ① ② ③ 步骤均标 ✓
- compliance 行显示 `100.0% (9/9 steps)`
- 无 REGRESSION DETECTED 字样
