# Scenario

`harness.sh worktree-clean REQ-037` を実行したとき、worktree が正常に削除される。
また、worktree が存在しない場合も exit 0 で終了すること（幂等性）。

# Steps

1. 事前準備：`./scripts/harness.sh implement REQ-037` で worktree を作成
2. `git worktree list` で worktree の存在を確認
3. `./scripts/harness.sh worktree-clean REQ-037` を実行
4. `git worktree list` で worktree が除去されたことを確認
5. 再度 `./scripts/harness.sh worktree-clean REQ-037` を実行（worktree なし状態）
6. 終了コードを確認

# Expected

- Step 4 で `git worktree list` に `~/workspace-menglan/open-workhorse` が表示されないこと
- Step 5（不存在時の再実行）が exit 0 で終了すること
- ログに「worktree 已移除」または「worktree 不存在，跳过」が出力されること
