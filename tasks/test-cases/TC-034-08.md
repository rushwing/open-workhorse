# Scenario

handler 失败后，`failed/` 中的文件必须：保留原 frontmatter，且在末尾追加 `ERROR:` 摘要行。

# Steps

1. 设置 SHARED_RESOURCES_ROOT 为临时目录
2. 在 `for-pandas/pending/` 写入含完整 frontmatter 的 ATM notification 消息
3. Override `_dispatch_msg()` 为 `return 1`
4. 调用 `inbox_init; inbox_read_pandas`
5. 读取 `for-pandas/failed/` 中的文件内容

# Expected

- 文件包含 `message_id: msg_034_08`（原 frontmatter 保留）
- 文件以 `---` 开头
- 文件包含 `ERROR:` 行
- `ERROR:` 行位于原 frontmatter 之后（追加，非替换）
