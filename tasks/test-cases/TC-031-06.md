# Scenario

`archive_merged_reqs()` 在 REQ 已经被归档（文件已在 `tasks/archive/done/`，
`tasks/features/` 中无对应文件）的情况下重复执行，不报错、不产生重复 commit。

# Steps

1. 在 tmpdir 创建：
   - `tasks/archive/done/REQ-099.md`（status: done）— 已归档
   - `tasks/features/` 目录为空（无 status: review 文件）
2. 注入 mock `gh`（期望不被调用）
3. 注入 mock `git`（期望不被调用）
4. 调用 `archive_merged_reqs()`

# Expected

- 函数正常退出（exit code 0）
- `tasks/archive/done/REQ-099.md` 存在且未被修改
- mock `gh` 未被调用
- mock `git` 未被调用（无多余 commit）
- 无错误输出
