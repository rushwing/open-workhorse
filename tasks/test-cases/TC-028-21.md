# Scenario

user_bug 本地 status 发生变更后，Pandas 同步流程将新状态以 `status:<local_status>` label 的形式推送到对应 GitHub issue，同时移除旧的 status:* label。

per bug-standard.md §8.2 step ②：每次同步时读取本地 status，与 GitHub issue 现有 status:* label 比对；若不一致则移除旧 label，添加新 label。

# Steps

1. 构造 bug fixture：`bug_type=user_bug, status=in_progress, github_issue=42`
2. Mock GitHub issue #42 现有 labels 为 `["bug-tracked", "status:confirmed"]`
3. Mock `gh.issueRemoveLabel` 和 `gh.issueAddLabel` 函数
4. 运行 GitHub 同步流程（status 推送步骤）
5. 断言 `gh.issueRemoveLabel(42, "status:confirmed")` 被调用（移除旧 label）
6. 断言 `gh.issueAddLabel(42, "status:in_progress")` 被调用（添加新 label）
7. 构造第二个场景：本地 status=in_progress，GitHub 已有 label `status:in_progress`（状态一致）
8. 再次运行同步
9. 断言 `gh.issueRemoveLabel` 和 `gh.issueAddLabel` 在第二次同步中**均未被调用**（状态一致时不重复操作）

# Expected

- 本地 status 与 GitHub label 不一致时：移除旧 status:* label，添加新 status:<local_status> label
- 本地 status 与 GitHub label 已一致时：不重复调用 label API（幂等）
- `bug-tracked` 等非 status:* label 不受影响
