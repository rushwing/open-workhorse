# Scenario

使用 `--update-baseline` flag 调用脚本时，无论当前 compliance 值如何，
应将当前结果写入 `eval/baseline.json` 并 exit 0。

# Steps

1. mock git logs 使 compliance=77.8%（7/9 步通过）
2. 现有 baseline.json 为 `{"compliance": 1.0}`
3. 调用 `eval:compliance --update-baseline`
4. 读取 baseline.json 确认更新
5. 检查退出码

# Expected

- 退出码为 0（--update-baseline 优先，不触发 REGRESSION DETECTED 的 exit 1）
- `eval/baseline.json` 内容更新为 `{"compliance": 0.778}` 或等价精度
- 控制台或 last-run.md 中有更新确认信息（如 "Baseline updated"）
