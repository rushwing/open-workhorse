# Scenario

`harness.sh implement REQ-037` が呼ばれたとき、git worktree が自動的に
`~/workspace-menglan/open-workhorse/` に作成される（新規作成ケース）。

# Steps

1. 事前確認：`git worktree list` を実行し、`~/workspace-menglan/open-workhorse` が存在しないことを確認
2. `./scripts/harness.sh implement REQ-037` を実行（Claude 呼び出し前のセットアップ部分だけ検証する場合は `FORCE=true` で dry-run 相当の確認が可能）
3. `git worktree list` を再度実行し、worktree が追加されていることを確認
4. `git -C ~/workspace-menglan/open-workhorse branch --show-current` を実行し、ブランチを確認

# Expected

- `git worktree list` の出力に `~/workspace-menglan/open-workhorse` が含まれる
- ブランチが `feat/REQ-037` であること
- スクリプトが exit 0 で終了すること
