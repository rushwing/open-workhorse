# 测试目标

验证 `claim_review_ready()` 函数正确扫描 `status: review_ready` 的 REQ，
将其原子转换为 `req_review`（`owner: huahua`），并向 Huahua inbox 写入评审请求。
这是 BUG-004 的回归测试用例。

# 前置条件

- `tasks/features/` 中存在至少一个 `status: review_ready`、`owner: unassigned` 的 REQ
- `SHARED_RESOURCES_ROOT` 指向可写临时目录
- `REPO_ROOT` 指向隔离的 tmpDir

# 测试步骤

1. 在 tmpDir 创建 `tasks/features/REQ-903.md`，设置 `status: review_ready`、`owner: unassigned`
2. 执行 `source pandas-heartbeat.sh; inbox_init; claim_review_ready`
3. 读取 `REQ-903.md` 验证 frontmatter 已更新
4. 读取 `inbox/for-huahua/pending/` 验证消息已写入

# 验收标准

- [ ] `REQ-903.md` 的 `status` 变为 `req_review`
- [ ] `REQ-903.md` 的 `owner` 变为 `huahua`
- [ ] `inbox/for-huahua/pending/` 中存在至少一个 `.md` 文件
- [ ] 该消息包含 `req_id: REQ-903`
- [ ] 脚本以 exit code 0 结束

# 边界场景

| 场景 | 预期行为 |
|---|---|
| `status: req_review`（已处理）| 不重复更新 |
| `status: review_ready` + `owner: pandas`（非 unassigned）| 不触发转换 |
| hold 标志文件存在 | 跳过，直接返回 |
| 无 `review_ready` REQ | 正常退出，无修改 |
