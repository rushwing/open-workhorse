# Scenario

`docs/req-flow.html` 应存在，包含全部 9 个状态节点关键词，并包含泳道图设计关键语义（
pending_bugs 路由信号、state-as-lock 交接语义、Bug clean 门控、owner 耦合角色标注），
确保可视化文档覆盖 v0.4 规程的核心契约，不能因只保留状态节点词而退化为无意义的关键词列表。

# Steps

**Group A — 文件存在性**
1. `test -f docs/req-flow.html`

**Group B — 9 个状态节点**
2. 逐一 grep 以下关键词（每次 `grep -q '<keyword>' docs/req-flow.html`）：
   `draft`, `review_ready`, `req_review`, `ready`, `test_designed`,
   `in_progress`, `blocked`, `review`, `done`

**Group C — 关键泳道语义标签**
3. `grep -q 'state-as-lock' docs/req-flow.html`（Pandas 原子 claim 语义）
4. `grep -q 'pending_bugs' docs/req-flow.html`（routing 信号字段）
5. `grep -q 'Bug clean' docs/req-flow.html`（review→done bug-clean 门控）
6. `grep -q 'Huahua' docs/req-flow.html`（owner-coupled 角色标注）
7. `grep -q 'Menglan' docs/req-flow.html`（owner-coupled 角色标注）
8. `grep -q 'Pandas' docs/req-flow.html`（入口/出口锚点标注）

**Group D — 无外部依赖**
9. `grep -q 'http://' docs/req-flow.html || grep -q 'https://' docs/req-flow.html`
   期望：exit 1（即不含外部 CDN 引用）

# Expected

- Group A：exit 0
- Group B：全部 9 个关键词命中（每次 grep exit 0）
- Group C：全部 6 个语义标签命中（每次 grep exit 0）
- Group D：无外部 CDN 引用（grep exit 1）
