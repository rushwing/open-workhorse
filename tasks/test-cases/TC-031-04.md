# Scenario

当 REQ `test_case_ref` 包含多个 TC 时，`archive_merged_reqs()` 将所有关联 TC 一并归档，
且每个 TC 的 frontmatter `status` 字段均更新为 `done`（BUG-005 修复覆盖）。

# Steps

1. 在 tmpdir 创建：
   - `tasks/features/REQ-099.md`（status: review，pr_number: 42，test_case_ref: [TC-099-01, TC-099-02, TC-099-03]）
   - `tasks/test-cases/TC-099-01.md`（status: ready）
   - `tasks/test-cases/TC-099-02.md`（status: ready）
   - `tasks/test-cases/TC-099-03.md`（status: ready）
2. 注入 mock `gh`：返回 `{"state":"MERGED"}`
3. 注入 mock `git`：记录调用
4. 调用 `archive_merged_reqs()`

# Expected

- `tasks/archive/done/TC-099-01.md` 存在，frontmatter `status: done`
- `tasks/archive/done/TC-099-02.md` 存在，frontmatter `status: done`
- `tasks/archive/done/TC-099-03.md` 存在，frontmatter `status: done`
- `tasks/test-cases/TC-099-01.md` 不存在
- `tasks/test-cases/TC-099-02.md` 不存在
- `tasks/test-cases/TC-099-03.md` 不存在
- `tasks/archive/done/REQ-099.md` 存在，frontmatter `status: done`
- git commit 被调用一次（REQ + 所有 TC 在同一 commit 中）
