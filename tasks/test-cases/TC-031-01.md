# Scenario

`archive_merged_reqs()` 检测到 `status: review` 的 REQ 对应 PR 已 merged，自动完成：
1. REQ frontmatter `status → done`
2. 关联 TC frontmatter `status → done`
3. 移动 REQ 文件至 `tasks/archive/done/`
4. 移动 TC 文件至 `tasks/archive/done/`
5. 执行 archive commit
6. 发送 Telegram 通知

# Steps

1. 在 tmpdir 创建仿库结构：
   - `tasks/features/REQ-099.md`（status: review，pr_number: 42，test_case_ref: [TC-099-01]）
   - `tasks/test-cases/TC-099-01.md`（status: ready）
   - `tasks/archive/done/`（目录存在）
2. 注入 mock `gh`（PATH 前置 tmpdir/bin）：`gh pr view 42 --json state` 返回 `{"state":"MERGED"}`
3. 注入 mock `git`：记录调用参数
4. 注入 mock `tg_notify`（stub 函数）
5. 调用 `archive_merged_reqs()`

# Expected

- `tasks/archive/done/REQ-099.md` 存在，且 frontmatter `status: done`
- `tasks/archive/done/TC-099-01.md` 存在，且 frontmatter `status: done`
- `tasks/features/REQ-099.md` 不存在
- `tasks/test-cases/TC-099-01.md` 不存在
- git commit 被调用，message 匹配 `archive(REQ-099): move to tasks/archive/done/`
- tg_notify 被调用至少一次，含 "REQ-099" 字样
