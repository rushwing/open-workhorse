# Scenario

Bug 的 review_round 字段包含非非负整数值（如负数或非数字字符串），字段校验器应返回错误。

per check-bug-coverage.sh：review_round 存在时必须为非负整数（`^[0-9]+$`）。

# Steps

1. 调用 `validateBugFields({review_round: "-1"})`；断言 `errors` 含 review_round 相关错误
2. 调用 `validateBugFields({review_round: "abc"})`；断言 `errors` 含 review_round 相关错误
3. 调用 `validateBugFields({review_round: "1.5"})`；断言 `errors` 含 review_round 相关错误
4. 调用 `validateBugFields({review_round: "0"})`；断言 `errors` 为空（0 是合法值）
5. 调用 `validateBugFields({review_round: "3"})`；断言 `errors` 为空（正整数合法）
6. 调用 `validateBugFields({})`（review_round 字段缺失）；断言 `errors` 为空（字段可选）

# Expected

- review_round 为负数、非数字字符串、小数时：返回至少 1 个校验错误，错误指向 review_round 字段
- review_round 为 "0" 或正整数字符串时：校验通过（errors 为空）
- review_round 字段缺失时：校验通过（字段可选，仅在存在时验证格式）
