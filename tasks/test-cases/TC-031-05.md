# Scenario

当 `gh pr view` 调用失败（模拟网络/权限错误），`archive_merged_reqs()` 应：
- 跳过该 REQ（不归档，不报错退出）
- 记录 WARN 日志
- 继续处理其他 REQ（心跳不中断）

# Steps

1. 在 tmpdir 创建两个 REQ：
   - `tasks/features/REQ-098.md`（status: review，pr_number: 41）— gh 调用将失败
   - `tasks/features/REQ-099.md`（status: review，pr_number: 42，test_case_ref: [TC-099-01]）— gh 调用成功
   - `tasks/test-cases/TC-099-01.md`（status: ready）
2. 注入 mock `gh`：
   - `pr view 41` 返回 exit code 1（stderr: "GraphQL: Not Found"）
   - `pr view 42` 返回 `{"state":"MERGED"}`
3. 调用 `archive_merged_reqs()`

# Expected

- `archive_merged_reqs()` 退出码为 0（不中断）
- `tasks/features/REQ-098.md` 仍存在（跳过，未归档）
- 标准错误或日志中含 WARN 字样及 "REQ-098"
- `tasks/archive/done/REQ-099.md` 存在，status: done（REQ-099 正常归档）
- `tasks/archive/done/TC-099-01.md` 存在，status: done
