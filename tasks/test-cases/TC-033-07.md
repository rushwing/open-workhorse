# Scenario

`inbox_read_pandas()` 处理 ATM Envelope 格式（type=response, legacy_type=dev_complete）消息时，
`status: completed`（ATM canonical）被正确识别为成功路径，触发 `tg_pr_ready`。

验证 BLOCK-1 修复：`status=completed` 不应走 dev_complete(blocked) 分支。

# Steps

1. 在 `for-pandas/` 写入 ATM response 消息：
   ```yaml
   ---
   type: response
   from: menglan
   to: pandas
   ...
   ---
   status: completed
   legacy_type: dev_complete
   pr_number: 42
   ```
2. 调用 `inbox_read_pandas`（mock tg_pr_ready）

# Expected

- exit code 0
- `tg_pr_ready` 被调用（completed = success path）
- 不触发告警（dev_complete(blocked) 分支不走）
- 消费后消息文件被删除
