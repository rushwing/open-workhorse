# Goal

将 open-workhorse 的核心 Agent 心跳脚本（bash）迁移至 TypeScript，以实现：

1. **可测试性**：用真正的单元测试（mock I/O、mock child_process）替代黑盒 bash subprocess 测试
2. **类型安全**：ATM Envelope、REQ frontmatter、inbox 操作全部有类型保护，编译期暴露错误
3. **可维护性**：逻辑可 review、可 diff、可静态分析，Hua Hua review 成本大幅降低
4. **与现有 TS 基础设施集成**：直接 import `src/runtime/` 模块，不再通过 grep/sed 解析 markdown

## 迁移范围

| 脚本 | 行数 | 子 REQ |
|------|------|--------|
| `scripts/pandas-heartbeat.sh` | ~1315 行 | REQ-042（共享基础层）+ REQ-043（Pandas 心跳逻辑） |
| `scripts/menglan-heartbeat.sh` | ~362 行 | REQ-044 |
| `scripts/huahua-heartbeat.sh` | ~518 行 | REQ-045 |
| `scripts/harness.sh` | ~754 行 | REQ-046 |

## 迁移策略

- **不改变外部接口**：cron 调用方式、inbox 目录结构、消息格式（ATM Envelope）全部保持不变
- **同名 CLI 入口保留**：bash wrapper 仅保留作为 `node --import tsx scripts/xxx.ts "$@"` 的薄包装，或直接重命名 npm scripts
- **分阶段交付**：按子 REQ 逐个迁移，每个子 REQ 独立可测、独立合入
- **旧 bash 脚本**：子 REQ done 后保留原 `.sh` 文件并标注 `@deprecated`，待 Umbrella done 后统一删除

## 验收标准

- 子 REQ-042 ~ REQ-046 全部 done
- `npm run build` 通过

- `npm run release:audit` 通过
- cron 重新注册后，系统端到端行为（inbox 收发、heartbeat 触发、Telegram 通知）与重构前一致
- 旧 `.sh` 文件标注 `@deprecated` 并保留至 Umbrella done

## Agent Notes

### 为什么先做重构再做 REQ-026

REQ-026 要往 `pandas-heartbeat.sh` 加约 200+ 行新逻辑（umbrella 检测、PR merge 检测、批量状态流转）。在现有 1315 行 bash 文件上继续堆代码，会让测试更难写、review 更难做。先完成 REQ-042+REQ-043，再在 TypeScript 版本上实现 REQ-026，可以从一开始就获得真正的单元测试覆盖。

### Node.js 调用的 LLM 便利性

与 bash 调用方式一致：`node --import tsx scripts/pandas-heartbeat.ts` 或通过 npm scripts 包装。LLM 感知不到底层变化。

### 现有 TS 基础设施可复用

- `src/runtime/task-store.ts`：任务状态读写
- `src/runtime/project-store.ts`：项目状态
- ATM Envelope 类型定义可新建 `src/contracts/inbox-envelope.ts`
- REQ frontmatter 解析可新建 `src/contracts/req-frontmatter.ts`
