# Goal

验证 openclaw CLI 能够加载指定 workspace（Lion），并使 Telegram Bot 正常上线、可收发消息，为后续 Agent 场景提供通信基础。

# In Scope

- 确认 `openclaw.json` 中 Lion workspace 配置正确
- 启动 openclaw 进程并加载 Lion workspace
- 通过 Telegram 客户端发送一条消息，Bot 能够响应（哪怕是 echo 或默认回复）
- 记录验证结果（日志截图或文本）

# Out of Scope

- Bot 回复内容的质量或 LLM 接入（REQ-003/REQ-004 覆盖）
- Telegram webhook vs polling 模式的优化
- open-workhorse UI 的集成展示

# Acceptance Criteria

- `openclaw` 启动成功，无 fatal 错误
- Lion workspace 的 Telegram Bot token 有效，Bot 状态为 online
- 发送一条测试消息后，Bot 在 60 秒内有响应（任意内容）
- 验证过程有日志或截图留存

# Test Case Design Notes

需要覆盖的场景：
- TC: openclaw 正常启动，workspace 加载成功（无 crash、无 auth error）
- TC: Telegram Bot 可接收消息（sendMessage API 调用成功）
- TC: Bot 响应消息（至少收到回复，内容不限）

注意：测试依赖真实的 Telegram Bot token 和 openclaw binary，属于 L3 集成测试。

# Agent Notes

- 认领前确认 `openclaw` binary 已在 PATH，`openclaw.json` 已配置 Lion workspace
- 若 token 缺失，在此处记录 blocked 原因并等待 Daniel 提供
- 验收依赖外部 Telegram 网络，Pi 环境需先完成 REQ-007（代理初始化）
