# Goal

在现有 `scripts/pandas-heartbeat.sh` 的 inbox 机制上，新增统一消息 Envelope 语义层。

**不改变目录结构，不改变传输机制（cron + 文件系统）。**
只在 frontmatter 层面做扩展，是最小侵入的语义升级。

这是 ATM（Agent Teams Messaging）协议的 P0 Sprint，为后续 Correlation / Claim 机制打基础。

# In Scope

- `scripts/pandas-heartbeat.sh`：新增 `inbox_write_v2()` 函数，支持新 Envelope 字段
- `scripts/pandas-heartbeat.sh`：`inbox_read_pandas()` 兼容新旧格式（按字段是否存在判断）
- 新 Envelope 必填字段：`message_id`、`type`（request|response|notification）、`from`、`to`、`created_at`、`thread_id`、`correlation_id`
- type=request 附加字段：`action`、`response_required`
- type=response 附加字段：`in_reply_to`
- type=notification 附加字段：`event_type`
- 旧有 `inbox_write()` 函数保留，标注 `@deprecated`，内部调用 `inbox_write_v2()` 以保证向后兼容
- `message_id` 生成规则：`msg_{from}_{yyyymmddHHMMSS}_{rand4}`
- `thread_id` 生成规则：`thread_{req_id}_{epoch}`
- `correlation_id` 生成规则：`corr_{req_id}_{epoch}`
- `harness/inbox-protocol.md` 新建，集中定义 Envelope 规范
- `tasks/test-cases/TC-033-*.md` 新建（10 个 TC）

# Out of Scope

- 目录结构变更（pending/claimed/done/failed 子目录 → REQ-034）
- Claim 原子性机制 → REQ-034
- Thread/Correlation 持久化追踪 → REQ-035
- 文件命名规范变更 → REQ-036
- menglan-heartbeat.sh / huahua-heartbeat.sh 中的 inbox 调用迁移

# Acceptance Criteria

- `inbox_write_v2()` 存在，签名：`<target> <type> <action_or_event> <thread_id> <correlation_id> <in_reply_to> <priority> <response_required> <payload_file>`
- `inbox_write_v2()` 生成的文件 frontmatter 包含所有必填 Envelope 字段
- `inbox_write()` 调用 `inbox_write_v2()` 并传入对应参数，旧调用不破坏
- `inbox_read_pandas()` 能解析新 Envelope 格式（按 `type` 字段值路由 handler）
- `inbox_read_pandas()` 仍能解析旧格式（`type: implement` 等旧值映射，`type` 缺失时 warn + skip）
- 新增对 `type: notification` 的处理分支（severity=action-required 时触发 tg_notify）
- `npm run build` 通过
- `npm test` 通过，新增覆盖新格式解析和旧格式兼容的单元测试

# Test Case Design Notes

需覆盖以下场景：

- TC-033-01：`inbox_write_v2()` 生成文件 — 必填 Envelope 字段全部存在且格式正确
- TC-033-02：`inbox_write_v2()` type=request — action / response_required 字段存在
- TC-033-03：`inbox_write_v2()` type=response — in_reply_to 字段存在
- TC-033-04：`inbox_write_v2()` type=notification — event_type 字段存在
- TC-033-05：`inbox_write()` 旧函数调用不破坏现有行为（backward compat）
- TC-033-06：`inbox_read_pandas()` 读新格式 request → 路由到正确 handler
- TC-033-07：`inbox_read_pandas()` 读新格式 response → 路由到正确 handler
- TC-033-08：`inbox_read_pandas()` 读 notification severity=action-required → tg_notify 触发
- TC-033-09：`inbox_read_pandas()` 读旧格式（type: implement）→ 旧 handler 正常处理
- TC-033-10：`inbox_read_pandas()` 读旧格式（type: dev_complete）→ 旧 handler 正常处理
