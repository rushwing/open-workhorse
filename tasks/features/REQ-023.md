# Goal

实现 TC 设计闭环：Pandas 委派 Huahua 设计 TC，Menglan 通过 `harness.sh tc-review` 评审，最多 2 轮迭代；超出上限则通过 Telegram 升级决策。

# In Scope

- `harness.sh tc-review <PR#>`：调用 Claude 以 adequate/missing-branch/redundant 标签评审 TC
- `_handle_tc_complete`：处理 `tc_complete` 消息，路由到 menglan（success）或 huahua（blocked，iteration+1）
- iteration ≥ 2 时调用 `tg_decision` 升级而非再次路由到 Huahua
- `harness.sh tc-review` 无参数时输出用法错误并 exit≠0

# Out of Scope

- Huahua 设计 TC 的具体实现
- TC PR 合并逻辑

# Acceptance Criteria

- `bash scripts/harness.sh tc-review`（无参数）exit≠0，stderr 含 "用法" 或 "tc-review"
- `tc_complete` status=success → for-menglan/ 出现 `type: implement` 消息
- `tc_complete` status=blocked iteration=1 → for-huahua/ 出现 `iteration: 2` 的 tc_design 消息
- `tc_complete` status=blocked iteration=2 → 调用 `tg_decision`，for-huahua/ 无新文件

# Agent Notes

- `_handle_tc_complete` 实现在 `scripts/pandas-heartbeat.sh`
- `cmd_tc_review` 实现在 `scripts/harness.sh`（从三个来源提取 REQ hint：branch name > PR title > changed files）
