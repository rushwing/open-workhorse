# Goal

实现 TC 设计闭环：Pandas 委派 Huahua 设计 TC，Menglan 通过 `harness.sh tc-review` 评审，
最多 2 轮迭代后批准，Pandas 再委派 Menglan 实现。

设计细节见 `docs/PANDAS-ORCHESTRATION.md` §7。

# In Scope

- `harness.sh tc-review <PR#>` 子命令（新增）
  - 获取 TC PR 的 review comments（与 fix-review 相同的 gh api 调用）
  - 注入 Menglan 评审 prompt（adequate / missing-branch / redundant 标签）
  - 结论行：`tc-review: APPROVED` 或 `tc-review: NEEDS_CHANGES`
- `scripts/pandas-heartbeat.sh` 中的 TC 路由逻辑：
  - `tc_complete` 消息（status=success）→ 写 `inbox/for-menglan/`: implement REQ-N
  - `tc_complete` 消息（status=blocked）且迭代 ≤ 2 → 写 `inbox/for-huahua/`: fix TC PR
  - 超过 2 轮 → `tg_decision`

# Out of Scope

- Huahua 实际执行 TC 设计的逻辑（agent 内部行为）
- code review 循环（REQ-020 整体流程，非本 REQ）

# Acceptance Criteria

- `./scripts/harness.sh tc-review 42` 执行后调用 Claude Code 并传入 TC 评审 prompt
- prompt 包含 PR #42 的 review comments 和 REQ acceptance criteria 指引
- prompt 要求每个 TC 标注 adequate / missing-branch / redundant
- 迭代计数从 inbox 消息中读取；>2 时触发 tg_decision 而非继续循环
- `bash -n scripts/harness.sh` 无语法错误
- `npm run release:audit` 通过
- `npm run req:check` 通过

# Test Case Design Notes

- TC-023-01: `harness.sh tc-review` 无参数时输出用法错误
- TC-023-02: `harness.sh tc-review 99`（有效 PR）调用 Claude Code（mock claude binary）
- TC-023-03: tc_complete status=success → inbox/for-menglan/ 出现 implement 消息
- TC-023-04: tc_complete status=blocked iteration=1 → inbox/for-huahua/ 出现 fix 消息
- TC-023-05: tc_complete status=blocked iteration=2 → 调用 tg_decision

# Agent Notes

- `cmd_tc_review` 模式与 `cmd_fix_review` 高度相似，可直接复用 gh api 调用部分
- 迭代计数存储在 inbox 消息的 frontmatter 中（新增 `iteration: N` 字段）
- `tg_decision` 在 REQ-024 实现前，可用 `warn` 打印占位；实现后替换
