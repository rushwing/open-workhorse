# Goal

确保 open-workhorse 在 `MONITOR_CONTINUOUS=true` 环境下，`/healthz` 接口持续返回 `{status:"ok"}`，为 Pi 部署的健康监控提供可靠基线。

# In Scope

- 验证 `MONITOR_CONTINUOUS=true` 时监控进程持续运行（不在首次 tick 后停止）
- 验证 `/healthz` 响应体包含 `status: "ok"` 字段
- 验证超过 60 秒后接口仍返回 `ok`（非 `stale`）

# Out of Scope

- UI 页面的健康状态显示
- 监控其他服务（仅 open-workhorse 自身）
- 告警或通知机制

# Acceptance Criteria

- `MONITOR_CONTINUOUS=true npm run dev:ui` 启动后，`curl /healthz` 返回 `{"status":"ok"}`
- 等待 90 秒后再次请求 `/healthz`，仍返回 `{"status":"ok"}`（非 stale）
- `npm test` 中有覆盖此逻辑的测试用例通过

# Test Case Design Notes

需要覆盖的场景：
- TC: MONITOR_CONTINUOUS=true — 启动 90 秒后 /healthz 返回 ok
- TC: MONITOR_CONTINUOUS 未设置（或 false）— /healthz 返回 stale 或 error（负向验证）
- TC: 单元测试 — monitor tick 逻辑在 continuous 模式下循环调用

注意：90 秒等待可在 L2 集成测试中用时间 mock 代替，避免 CI 超时。
