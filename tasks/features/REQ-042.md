# Goal

为后续心跳脚本的 TypeScript 迁移提供共享基础层：

1. **ATM Envelope 类型定义**：将 `harness/inbox-protocol.md` 中的 Envelope 规范转化为 TypeScript 类型，作为所有心跳脚本的共享契约
2. **REQ Frontmatter 解析**：取代现有各脚本中重复的 `_get_fm_field()` bash 函数，提供类型安全的 markdown frontmatter 读写
3. **Inbox I/O 操作**：取代 `inbox_write_v2()`、`inbox_read_*()` 等 bash 函数，提供可 mock、可测试的文件 I/O 操作

## 背景

当前四个 bash 脚本（pandas/menglan/huahua-heartbeat.sh、harness.sh）各自复制了类似的 `_get_fm_field()` 和 inbox 操作。这是技术债的根源：逻辑分散、难以单元测试、类型缺失。

本 REQ 提取这些共享原语，是 REQ-043~046 的前置依赖。

## In Scope

### 1. `src/contracts/inbox-envelope.ts`

ATM Envelope 类型（对应 `harness/inbox-protocol.md §2`）：

```typescript
// 基础 Envelope
interface BaseEnvelope {
  message_id: string;        // msg_{from}_{yyyymmddHHMMSS}_{rand4}
  type: 'request' | 'response' | 'notification';
  from: AgentName;
  to: AgentName;
  created_at: string;        // ISO 8601 UTC
  thread_id: string;
  correlation_id: string;
  priority: 'P0' | 'P1' | 'P2' | 'P3';
}

// request 变体
interface RequestEnvelope extends BaseEnvelope {
  type: 'request';
  action: RequestAction;     // 枚举：implement | req_review | review | bugfix | tc_design | ...
  response_required: boolean;
  objective: string;
  scope: string;
  expected_output: string;
  done_criteria: string;
  context_summary?: string;
  references?: Reference[];
  delegation_incomplete?: boolean;
}

// response 变体
interface ResponseEnvelope extends BaseEnvelope {
  type: 'response';
  in_reply_to: string;
  status: ResponseStatus;    // completed | partial | blocked | failed | rejected | deferred
  summary?: string;
}

// notification 变体
interface NotificationEnvelope extends BaseEnvelope {
  type: 'notification';
  event_type: string;
  severity: 'info' | 'warn' | 'action-required';
}

type Envelope = RequestEnvelope | ResponseEnvelope | NotificationEnvelope;
```

ID 生成函数：`generateMessageId(from)`、`generateThreadId(reqId)`、`generateCorrelationId(reqId)`

### 3. `src/lib/inbox-io.ts`

```typescript
// 初始化 inbox 目录结构（幂等）
function inboxInit(sharedResourcesRoot: string): Promise<void>

// 写入消息到 pending/（原子写）
function inboxWrite(sharedResourcesRoot: string, to: AgentName, envelope: Envelope): Promise<string>  // 返回文件路径

// 原子 claim：pending/ → claimed/（返回 claimed 文件路径，无消息时返回 null）
function inboxClaim(sharedResourcesRoot: string, agent: AgentName): Promise<{path: string; envelope: Envelope} | null>

// 处理完成：claimed/ → done/
function inboxFinalize(claimedPath: string, result: 'done' | 'failed', errorSummary?: string): Promise<void>

// 读取旧格式（_inbox_read_legacy 的 TS 对应）
function inboxReadLegacy(filePath: string): Promise<LegacyMessage | null>
```

所有 I/O 操作接受 `sharedResourcesRoot` 参数（可注入），方便测试时指向 tmpdir。

### YAML frontmatter 解析库选型

建议使用 `gray-matter`（MIT 许可，已广泛用于静态站点生成器）。确认 package.json 中是否已有，如无则需要先确认许可再添加。替代方案：自己用正则解析（成本高），或用 `js-yaml`（MIT）手动切割 frontmatter 区块。

### 并发安全

`inboxClaim` 的原子 mv 在 Linux 文件系统上由 `fs.rename` 保证原子性（同一文件系统）。如需跨文件系统，需要 copy + unlink 方案。当前 tmpdir 和 inbox 都在同一文件系统，`fs.rename` 足够。

### 类型与现有 types.ts 的关系

`src/types.ts` 已有 `ProjectTask`、`TaskState` 等 task-store 类型。新的 `inbox-envelope.ts` 和 `req-frontmatter.ts` 放在 `src/contracts/` 以区分"harness 合约"和"API 数据模型"。不要修改 `src/types.ts`，避免影响现有 API。
