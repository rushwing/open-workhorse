# Goal

通过 LLM ReAct 循环完成 open-workhorse systemd user service 注册，启动服务，并执行健康自验证——确认 `/healthz` 返回 `{status:"ok"}`。

# In Scope

- 创建并注册 `open-workhorse.service`（`~/.config/systemd/user/` 路径，user 级别管理）
- `systemctl --user start open-workhorse.service` 并设置 `systemctl --user enable`（开机自启）
- 验证 `systemctl --user status open-workhorse.service` 为 active (running)
- 验证 `curl http://localhost:4310/healthz` 返回 `{"status":"ok"}`

# Out of Scope

- 反向代理 / HTTPS 配置
- 外网暴露（仅 localhost 或 Tailscale 内网）
- 监控告警集成
- 服务日志轮转配置

# Acceptance Criteria

- `systemctl --user status open-workhorse.service` 显示 active (running)
- `curl http://localhost:4310/healthz` 返回 `{"status":"ok"}`（HTTP 200）
- 服务在 reboot 后自动启动（`systemctl --user enable` 已设置）

# Test Case Design Notes

需要覆盖的场景：
- TC: service 注册和启动（`systemctl --user status` active 验证）
- TC: /healthz 健康检查通过（REQ-005 在 Pi 上的集成验证）
- TC: reboot 后服务自动恢复（`systemctl --user enable` 验证）

注意：L3 硬件集成测试。本 REQ 是 P2 Agentic Deploy 序列的终点。
