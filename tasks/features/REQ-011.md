# Goal

通过 LLM ReAct 循环完成 open-workhorse systemd service 注册，启动服务，并执行健康自验证——确认 `/healthz` 返回 `{status:"ok"}`，整个 Agentic Deploy 序列（REQ-009 到 REQ-011）全程耗时在 15 分钟内。

# In Scope

- 创建并注册 `open-workhorse.service`（systemctl --user 或 system 级别）
- `systemctl start open-workhorse.service` 并设置 `enable`（开机自启）
- 验证 `systemctl status open-workhorse.service` 为 active (running)
- 验证 `curl http://localhost:4310/healthz` 返回 `{"status":"ok"}`
- 记录从 REQ-009 git clone 开始到本步骤完成的总耗时

# Out of Scope

- 反向代理 / HTTPS 配置
- 外网暴露（仅 localhost 或 Tailscale 内网）
- 监控告警集成
- 服务日志轮转配置

# Acceptance Criteria

- `systemctl status open-workhorse.service` 显示 active (running)
- `curl http://localhost:4310/healthz` 返回 `{"status":"ok"}`（HTTP 200）
- 服务在 reboot 后自动启动（enable 已设置）
- 从 git clone 到 /healthz ok 全程 < 15 分钟（记录时间戳）

# Test Case Design Notes

需要覆盖的场景：
- TC: service 注册和启动（active 状态验证）
- TC: /healthz 健康检查通过（REQ-005 在 Pi 上的集成验证）
- TC: reboot 后服务自动恢复（enable 验证）
- TC: 全程耗时记录（< 15 min）

注意：L3 硬件集成测试。本 REQ 是 P2 Agentic Deploy 序列的终点。
