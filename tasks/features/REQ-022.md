# Goal

将 `pandas-heartbeat.sh` 集成到 OpenClaw APP_COMMAND 机制，使 Pandas 编排循环可通过 heartbeat tick 触发。实现 auto_claim（优先级排序任务认领）和 stall_detection（停滞检测 + Telegram 告警）。

# In Scope

- `src/index.ts` 中 `APP_COMMAND=pandas-heartbeat` 分发到 `scripts/pandas-heartbeat.sh`
- `normalizeCommand("pandas-heartbeat")` 支持
- `auto_claim` 函数：按 status_tier（test_designed > ready）和 priority（P0>P1>P2）排序认领
- `stall_detection` 函数：读取 `.harness_sessions`，检测 `in_progress` 任务停滞超 `DEV_WATCHDOG_STALE_HOURS` 小时
- hold 模式下 auto_claim 跳过

# Out of Scope

- 具体 agent 实现逻辑
- cron 安装

# Acceptance Criteria

- `APP_COMMAND=pandas-heartbeat` 调用后无 "Unknown command" 错误
- empty inbox 时 exit 0，无 ERROR 输出
- P1 任务优先于 P2 任务被认领
- 超过阈值的 `in_progress` 任务触发 `tg_notify` 告警，消息含任务 ID 和 "停滞"

# Agent Notes

- `src/index.ts` 的 `runCommand` 和 `normalizeCommand` 函数已支持 `pandas-heartbeat`
- `auto_claim` 实现在 `scripts/pandas-heartbeat.sh`
- `stall_detection` 读取 `$REPO_ROOT/.harness_sessions`（可通过 `HARNESS_SESSION_LOG_OVERRIDE` 覆盖以便测试）
