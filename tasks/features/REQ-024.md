# Goal

实现 Daniel→Pandas 的文本指令解析（start/status/hold/resume）和 Pandas→Daniel 的 tg_decision（带 inline keyboard，超时后重通知）。

# In Scope

- `tg_poll_commands`：轮询 getUpdates，返回来自 CHAT_ID 的文本消息；未配置时静默返回
- `handle_telegram_commands`：解析 start/status/hold/resume 指令
- hold/resume 指令操作 `.pandas_hold` 文件
- `tg_decision`：发送带 inline keyboard 消息，轮询 callback；超时（默认 86400s）后调用 tg_notify 重通知并返回非零
- offset 持久化到 `.pandas_tg_offset`

# Out of Scope

- Telegram webhook 模式
- 多用户指令

# Acceptance Criteria

- `TELEGRAM_BOT_TOKEN` 未设置时 `tg_poll_commands` exit 0，无 ERROR
- `start REQ-021` 指令触发 `auto_claim_specific "REQ-021"`，REQ-021 owner 变为 claude_code
- `hold` 指令创建 `.pandas_hold` 文件
- `resume` 指令删除已存在的 `.pandas_hold` 文件
- `tg_decision` 在 `TG_DECISION_TIMEOUT=1` 超时后返回非零，stderr 含 "decision timed out"，调用 tg_notify

# Agent Notes

- `tg_poll_commands` 和 `tg_decision` 实现在 `scripts/telegram.sh`
- `handle_telegram_commands` 实现在 `scripts/pandas-heartbeat.sh`
- 测试时通过 mock `curl` 或 mock 函数覆盖网络调用
