# Goal

将现有单 Agent（claude_code）规程升级为多 Agent 协作模型，支持 Pandas / Huahua / Menglan 三个 Agent 的任务认领、TC 设计、代码审查、打回修复、合并通知全链路状态机。

需修订以下三个 harness standard 文件，填补系统性缺口：

1. `harness/requirement-standard.md` — owner 扩展 + blocked_reason + review→blocked + bug-gate + review_round
2. `harness/testing-standard.md` — TC 所有权流转章节（新增 §10）
3. `harness/bug-standard.md` — Bug→REQ blocking 规范（新增 §2.2）

# In Scope

- `harness/requirement-standard.md`：
  - `owner` 枚举扩展：`pandas / huahua / menglan` 加入合法值
  - `blocked_reason` 字段：新增，枚举 `dep_not_done | review_rejected | bug_linked | external_decision`
  - `review_round` 字段（可选）：追踪打回轮次
  - claim/handoff 协议：多 Agent 认领 + 移交规程
  - `review → blocked` 合法转换：新增（打回路径）
  - Bug clean → done 门控：Agent Notes 有 Bug 外链时的前置检查规则
  - review_round 上限：超 2 轮升级 Daniel
- `harness/testing-standard.md`：
  - 新增 §10：TC 所有权流转（Pandas → Huahua design → Menglan review → Menglan implement）
  - TC review 打回规程 + 2 轮上限
- `harness/bug-standard.md`：
  - 新增 §2.2：Bug→REQ blocking 规范（开 Bug 时联动 REQ）
  - 新增 §2.3：Bug clean → REQ unblock 规范（Bug 关闭时联动 REQ）
- `scripts/check-req-coverage.sh`：
  - OWNER_ENUM 扩展，加入 `pandas huahua menglan`

# Out of Scope

- 规程实现（pandas-heartbeat.sh 的代码变更，属 REQ-026 范围）
- IPC 消息格式定义（属 REQ-026 范围）
- UI 可视化

# Acceptance Criteria

- `harness/requirement-standard.md`：owner 枚举、blocked_reason、review→blocked、bug-gate、review_round 全部记录在案
- `harness/testing-standard.md`：TC 所有权流转章节存在，流程可被 Agents 独立遵循
- `harness/bug-standard.md`：Bug→REQ blocking 和 Bug clean→unblock 规范存在
- `scripts/check-req-coverage.sh`：OWNER_ENUM 包含 pandas/huahua/menglan
- `npm run req:check` 通过（REQ-027 frontmatter 合规）
