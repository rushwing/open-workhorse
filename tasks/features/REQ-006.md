# Goal

在 dev 机器（macOS）上构建 auto-mihomo 离线包（含所有依赖），并通过 SCP 经本地网络传输到目标树莓派，为后续离线安装步骤做准备。

# In Scope

- `build_package.sh` 脚本（在 auto-mihomo 项目或本 repo scripts/ 中）生成 `auto-mihomo-offline.tar.gz`
- SCP 传输脚本或操作手册（含目标路径、校验方式）
- tarball 内容完整性验证（md5/sha256 校验）

# Out of Scope

- mihomo 服务的实际安装和启动（REQ-007 覆盖）
- Tailscale 网络配置
- 对 auto-mihomo 上游代码的修改

# Acceptance Criteria

- `auto-mihomo-offline.tar.gz` 生成成功，大小合理（非空文件）
- SCP 传输到 Pi 成功，Pi 上文件 checksum 与 dev 机器一致
- 传输过程无需 Pi 联网（仅需本地网络 SSH 可达）

# Test Case Design Notes

需要覆盖的场景：
- TC: 本地构建 — build_package.sh 在 macOS 上运行成功，生成 tarball
- TC: SCP 传输 — 本地网络 SCP 成功，Pi 上文件存在且 checksum 正确
- TC: 构建产物验证 — tarball 包含 mihomo binary 和 upgrade.sh

注意：此为 L3 脚本集成测试，需要真实 Pi 硬件。
