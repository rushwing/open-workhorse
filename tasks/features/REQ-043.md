# Goal

将 `scripts/pandas-heartbeat.sh`（1315 行）迁移至 `scripts/pandas-heartbeat.ts`，保持**外部行为完全一致**，获得真正的单元测试覆盖。

这是整个重构链中最复杂的一步，包含系统中最多的状态机逻辑。

## In Scope

### 功能模块（对应 bash 脚本各区块）

| 模块 | bash 函数 | TS 对应 |
|------|-----------|---------|
| Inbox IPC | `inbox_init`、`inbox_write`、`inbox_write_v2` | 使用 REQ-042 `inboxWrite` |
| Inbox 读取 | `inbox_read_pandas`、`_inbox_read_legacy`、`_dispatch_msg` | `readAndDispatchPandasInbox()` |
| 消息处理器 | `_handle_dev_complete`、`_handle_review_complete`、`_handle_tc_complete` | `handleDevComplete()`、`handleReviewComplete()`、`handleTcComplete()` |
| Auto-claim | `auto_claim`、`auto_claim_specific`、`get_priority_rank` | `autoClaim()` |
| REQ review claim | `claim_review_ready` | `claimReviewReady()` |
| 重大决策检测 | `detect_major_decision`、`detect_major_decision_from_inbox` | `detectMajorDecision()` |
| Telegram 指令 | `handle_telegram_commands` | `handleTelegramCommands()` |
| Worktree 清理 | `_auto_worktree_clean` | `autoWorktreeClean()` |
| Post-merge 归档 | `archive_merged_reqs` | `archiveMergedReqs()` |
| 停滞检测 | `stall_detection`、`_check_stall_and_keepalive` | `stallDetection()` |
| 主入口 | `main()` | `main()` |

### 外部调用保留

以下外部工具调用通过 `child_process.spawn` 或 `execa` 封装，接口可 mock：

- `gh pr list`、`gh pr view`、`gh pr merge`（PR 状态检测）
- `git pull origin main`（merge 后同步）
- `bash scripts/harness.sh implement/bugfix/fix-review`（agent 调用，REQ-046 完成后可直接 import）
- Telegram Bot API（`tg_notify`、`tg_decision`）

### 配置注入

所有环境变量（`SHARED_RESOURCES_ROOT`、`REPO_ROOT`、`TELEGRAM_BOT_TOKEN`、`TELEGRAM_CHAT_ID` 等）通过 config 对象注入，测试时可覆盖。

## Out of Scope

- 修改任何业务逻辑或状态机行为
- umbrella REQ 编排逻辑（REQ-026，等本 REQ done 后再做）
- 修改 ATM Envelope 格式
- 删除 `pandas-heartbeat.sh`（Umbrella done 后统一删除）

## Agent Notes

### 迁移顺序建议

1. 先迁移 `inbox_read_pandas` + `_dispatch_msg`（消息路由框架）
2. 再迁移各消息处理器（逻辑最复杂）
3. 最后迁移 `auto_claim`、`stall_detection`、`main()`

### bash 版本的测试迁移

现有 `test/pandas-heartbeat.test.ts`（4075 行）测试通过 spawn bash 调用。迁移完成后，这些测试应逐步替换为直接调用 TS 函数的单元测试。旧测试可标注 `// @legacy-bash-test` 并在 REQ-043 完成后删除，但需确保新测试覆盖率不低于旧测试。

### hold 标志文件

`scripts/pandas-heartbeat.sh` 中的 hold 文件逻辑（`HOLD_FLAG`）需要一并迁移，不能遗漏。

### Telegram 封装

建议新建 `src/lib/telegram.ts`（在本 REQ 内完成），封装 `tgNotify()`、`tgDecision()`，测试时 mock 整个模块。
