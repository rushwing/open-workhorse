# Goal

验证 openclaw CLI 在启动时能够正确读取 Lion workspace 的 `AGENTS.md`（或等效 Agent 配置文件），并使 Agent 配置生效，为端到端对话场景做准备。

# In Scope

- 确认 Lion workspace 中 `AGENTS.md` 的路径和格式符合 openclaw 要求
- 启动 openclaw 并观测 Agent 配置是否被加载（通过日志、CLI 输出或 API 查询）
- 验证 Agent 名称、角色等关键字段从配置文件中正确读取

# Out of Scope

- `AGENTS.md` 内容的业务设计（该文件由 Daniel 维护）
- Agent 实际执行对话的质量（REQ-004 覆盖）
- 多 Agent 协作编排

# Acceptance Criteria

- openclaw 启动日志中包含 Lion workspace AGENTS.md 加载成功的标志
- 可通过 openclaw CLI 或日志确认至少一个 Agent（如 Lion）已注册
- 无因 AGENTS.md 格式错误导致的启动失败

# Test Case Design Notes

需要覆盖的场景：
- TC: 正常路径 — AGENTS.md 格式正确，openclaw 加载成功
- TC: 异常路径 — AGENTS.md 路径不存在时，openclaw 给出明确错误信息（而非 silent fail）
- TC: 字段验证 — Agent 名称等关键字段从文件中正确读取，非默认值

注意：属于 L3 集成测试，依赖 openclaw binary 和真实 workspace 文件。

# Agent Notes

- 认领前阅读 Lion workspace 中 AGENTS.md 的实际格式，确认字段名
- 若 openclaw 无 AGENTS.md 加载日志，可通过 openclaw list-agents 或等效命令验证
- 依赖 REQ-002（openclaw 基础启动）完成后认领
