# Goal

通过 LLM ReAct 循环在 Pi 上安装 `openclaw` binary，并完成 `openclaw.json` 配置（含 Lion workspace），使 openclaw CLI 可在 Pi 上正常运行。

# In Scope

- 安装 `openclaw` binary 到 PATH（通过代理 download 或 npm global install）
- 配置 `~/.openclaw/openclaw.json`（Lion workspace 配置，token 等）
- 配置 open-workhorse `.env` 文件（`OPENCLAW_HOME`、`LOCAL_API_TOKEN` 等关键变量）
- 验证 `openclaw version` 或等效命令可执行

# Out of Scope

- systemd service 注册（REQ-011 覆盖）
- openclaw.json 内容的业务设计（由 Daniel 维护）
- LLM API key 的管理策略

# Acceptance Criteria

- `which openclaw` 返回有效路径
- `openclaw version`（或 `openclaw --version`）输出版本号，退出码 0
- `cat ~/.openclaw/openclaw.json` 包含 Lion workspace 相关配置
- `.env` 文件存在，包含 `LOCAL_API_TOKEN`（非空）和 `OPENCLAW_HOME`

# Test Case Design Notes

需要覆盖的场景：
- TC: openclaw binary 安装（which openclaw 成功）
- TC: openclaw --version 执行成功
- TC: openclaw.json 内容验证（key 字段存在）
- TC: .env 关键字段验证

注意：L3 硬件集成测试。openclaw.json 内容（token、workspace id）为敏感信息，不提交到 repo。

# Agent Notes

- 依赖 REQ-009（open-workhorse 已 clone）
- openclaw binary 安装方式（npm i -g / 直接 download）在 ready 阶段确认
- `.env` 文件从 `.env.example` 复制后填入，敏感值由 Daniel 提供
- `LOCAL_API_TOKEN` 用 `openssl rand -hex 24` 生成
