# Goal

在树莓派上运行 `upgrade.sh` 安装 auto-mihomo，使 `mihomo.service` 处于 active 状态，Pi 的出站网络流量通过代理路由，为后续 git/curl/apt 操作提供网络基础。

# In Scope

- 在 Pi 上解压 `auto-mihomo-offline.tar.gz` 并执行 `upgrade.sh`
- 验证 `systemctl status mihomo.service` 显示 active (running)
- 验证通过代理访问外网（curl github.com 返回 200）
- 确认 `openclaw` 系统用户由 auto-mihomo 创建（供后续步骤使用）

# Out of Scope

- mihomo 配置的业务规则（订阅、规则集）— 由 Daniel 维护
- Tailscale 安装（REQ-008 覆盖）
- Pi 防火墙或路由规则的深度配置

# Acceptance Criteria

- `systemctl --user status mihomo.service` 或 `systemctl status mihomo.service` 显示 active
- `curl -x <proxy_addr> https://github.com` 返回非错误响应
- `id openclaw` 命令显示 openclaw 用户已存在

# Test Case Design Notes

需要覆盖的场景：
- TC: upgrade.sh 执行成功，无 fatal 错误
- TC: mihomo.service active 验证
- TC: 代理连通性验证（curl through proxy）
- TC: openclaw 用户存在性验证

注意：L3 硬件集成测试，需要真实 Pi。

# Agent Notes

- 依赖 REQ-006 完成（tarball 已传输到 Pi）
- upgrade.sh 路径在 auto-mihomo 包内
- 代理地址（默认 127.0.0.1:7890 或类似）需与 mihomo 配置一致
