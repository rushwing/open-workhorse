# Goal

完成 ATM 协议的 P2 Sprint，落实两个剩余规范：

1. **Delegation 结构化**：request 消息的任务委派内容必须结构化，
   不允许纯自然语言 body 作为任务规格
2. **文件命名规范**：新格式文件名携带路由信息，
   人和 agent 直接从文件名读取上下文

# In Scope

- `scripts/pandas-heartbeat.sh`：`inbox_write_v2()` 在 type=request 时校验 payload_file 中
  必须包含 `objective` / `scope` / `expected_output` / `done_criteria` 四个字段
  - 校验失败：warn 日志 + 仍写入文件，但在 envelope 中标记 `delegation_incomplete: true`
- 新文件命名规范：
  ```
  {timestamp}_{type}_{from}_to_{to}_{corr_or_evt}.md
  ```
- `inbox_read_pandas()` 兼容新旧两种文件命名

# Out of Scope

- open-workhorse UI 的 thread 链路视图
- 历史消息文件名批量重命名

# Acceptance Criteria

- `inbox_write_v2()` type=request 时，payload_file 缺少任一必填 delegation 字段 → warn + `delegation_incomplete: true`
- 新命名格式文件名可被 `inbox_read_pandas()` 正常解析
- 旧格式文件名仍可被正常解析（兼容 6 个月）
- `npm run build` 通过
- `npm test` 通过，覆盖委派字段校验和新旧文件名兼容场景

# Test Case Design Notes

需覆盖以下场景：

- TC-036-01：delegation 完整 — 四个字段全存在，无 warn，无 delegation_incomplete
- TC-036-02：delegation 不完整（缺 done_criteria）— warn + delegation_incomplete: true 写入
- TC-036-03：新命名格式 — inbox_read 正确解析并路由
- TC-036-04：旧命名格式 — inbox_read 正确解析并路由（兼容路径）
- TC-036-05：context_summary 超 500 字 — 自动截断 + warn
- TC-036-06：references 格式校验 — type 不在枚举内时 warn
