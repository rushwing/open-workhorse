# Goal

实现 Pandas 编排循环所需的文件系统 IPC 层：共享收件箱读写、消息格式校验、路径配置。

设计细节见 `docs/PANDAS-ORCHESTRATION.md` §3。

# In Scope

- `scripts/pandas-heartbeat.sh` 中的 inbox 初始化（目录不存在时自动创建）
- 收件箱写入函数：`inbox_write <target-agent> <type> <req_id> [pr_number] <summary> [blocking_reason]`
- 收件箱读取函数：扫描 `inbox/for-pandas/`，解析 YAML frontmatter，按 `type` 分发
- 文件命名格式：`YYYY-MM-DD-{from}-{description}.md`
- 处理完成后删除已消费文件
- `SHARED_RESOURCES_ROOT` 环境变量支持（默认值：`~/Dev/everything_openclaw/personas/shared-resources`）

# Out of Scope

- Telegram 通知（REQ-024）
- 心跳 tick 调度（REQ-022）
- TC 设计路由（REQ-023）

# Acceptance Criteria

- `inbox_write` 写出的文件包含合法 YAML frontmatter（type, req_id, summary, status 字段）
- `inbox_read_pandas` 正确解析 type 字段并分发到对应处理函数
- 处理完成的消息文件被删除（消费语义）
- inbox 根路径来自 `$SHARED_RESOURCES_ROOT`；未设置时使用默认值
- `bash -n scripts/pandas-heartbeat.sh` 无语法错误
- `npm run release:audit` 通过（无硬编码路径）
- `npm run req:check` 通过

# Test Case Design Notes

- TC-021-01: inbox_write 写出文件，断言文件存在且 frontmatter 合法
- TC-021-02: inbox_read_pandas 消费文件后，断言文件已删除
- TC-021-03: 设置 SHARED_RESOURCES_ROOT=/tmp/test-inbox，断言写入路径正确
- TC-021-04: SHARED_RESOURCES_ROOT 未设置时，断言使用默认路径（展开 ~）

# Agent Notes

- 优先实现 `inbox_write` 和 `inbox_read_pandas`；这两个函数是 REQ-022/023 的基础
- YAML frontmatter 解析可用 `awk`（与 harness.sh get_field 模式保持一致）
- 默认路径展开：`${SHARED_RESOURCES_ROOT:-$HOME/Dev/everything_openclaw/personas/shared-resources}`
