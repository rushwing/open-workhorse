# Goal

为 Pandas 编排循环建立基于文件系统的 IPC 机制（共享收件箱），实现 inbox_write / inbox_read_pandas 函数，支持 for-pandas / for-huahua / for-menglan 三个收件箱目录，消息格式符合 YAML frontmatter 规范。

# In Scope

- `inbox_write` 函数：写出消息到指定 agent 收件箱
- `inbox_read_pandas` 函数：读取并消费 for-pandas/ 中所有消息（删除已处理文件）
- 消息 YAML frontmatter 格式：type, req_id, summary, status, blocking_reason, iteration, pr_number
- `SHARED_RESOURCES_ROOT` 环境变量支持（显式设置与默认值）
- `inbox_init` 函数：确保三个收件箱目录存在

# Out of Scope

- 网络传输或远程 IPC
- 消息加密或访问控制

# Acceptance Criteria

- `inbox_write` 写出的文件首行为 `---`，包含 type/req_id/summary/status 字段
- `inbox_read_pandas` 处理后删除消息文件（消费语义）
- `SHARED_RESOURCES_ROOT` 显式设置时，写入路径在指定目录下
- 未设置 `SHARED_RESOURCES_ROOT` 时，使用 `$HOME/Dev/everything_openclaw/personas/shared-resources`，路径不含字面量 `~`
- `npm run req:check` 通过

# Agent Notes

实现位于 `scripts/pandas-heartbeat.sh` 的 `inbox_write`、`inbox_init`、`inbox_read_pandas` 函数。
文件名格式：`YYYY-MM-DD-pandas-{type}-{req_id}-{PID}-{RANDOM}.md`（含 PID+RANDOM 防止同日同类型碰撞）。
