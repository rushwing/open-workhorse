## Fixture REQ Design

```
eval/fixtures/
  REQ-E001.md   # tc_policy=exempt  вҖ” direct readyвҶ’in_progress path
  REQ-E002.md   # tc_policy=required вҖ” test_designedвҶ’in_progress path
  REQ-E003.md   # blocked scenario   вҖ” blockedвҶ’unblockedвҶ’in_progress path
```

Each fixture task is hello-world level (e.g., "create a shell script that prints hello").
This decouples task quality from harness compliance measurement. Agent completes in 5вҖ“10 min.
Fixtures do not change with codebase evolution.

1. `npm run eval:compliance` еҲҶжһҗ `feat/REQ-E0*` branchesпјҡ

   - compliance vs `eval/baseline.json` дёӢйҷҚ >10% ж—¶ exit 1 е№¶иҫ“еҮә `REGRESSION DETECTED`

2. `eval/last-run.md` еҢ…еҗ«пјҡ
   - жҜҸдёӘ fixture зҡ„ per-step з»“жһңиЎЁж јпјҲвң“ / вң— / - пјү
   - жҖ» compliance зҷҫеҲҶжҜ”пјҲйҖҡиҝҮжӯҘйӘӨж•° / жҖ»жӯҘйӘӨж•°пјү
   - дёҺ baseline еҜ№жҜ”зҡ„ delta еҖј
