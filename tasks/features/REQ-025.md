# Goal

实现重大决策检测机制（TRIGGER-001/002/003），当检测到外部凭证依赖、未知依赖阻塞或范围扩展时，通过 Telegram 升级决策给 Daniel。

# In Scope

- `detect_major_decision <req_file>`：扫描 REQ body，检测 TRIGGER-001（外部凭证依赖未在 .env.example 声明）
- `detect_major_decision_from_inbox`：处理 inbox 消息中的 TRIGGER-002（depends_on 阻塞）和 TRIGGER-003（范围扩展）
- 检测到触发条件时调用 `tg_decision` 通知 Daniel 并等待决策
- 无触发条件时函数返回 0

# Out of Scope

- 新增触发类型（TRIGGER-004+）
- tg_decision 超时处理（属 REQ-024）

# Acceptance Criteria

- `detect_major_decision` 对包含未声明 API_KEY 的 REQ 文件返回 1 并调用 `tg_decision`
- `detect_major_decision` 对 .env.example 中已声明的变量返回 0
- `blocking_reason` 含 "depends_on" 时触发 TRIGGER-002 → 调用 `tg_decision`
- `blocking_reason` 含 "outside REQ boundary" 或 "changes outside" 时触发 TRIGGER-003 → 调用 `tg_decision`
- 无触发条件（无凭证关键词，无阻塞原因匹配）时 `detect_major_decision` 返回 0，不调用 `tg_decision`

# Agent Notes

- `detect_major_decision` 和 `detect_major_decision_from_inbox` 实现在 `scripts/pandas-heartbeat.sh`
- `_handle_major_decision` 调用 `detect_major_decision_from_inbox` 处理 TRIGGER-002/003，未匹配时执行默认处理
- TRIGGER-001 在 `auto_claim` 和 `auto_claim_specific` 中调用 `detect_major_decision`，检测到触发条件时 skip 该任务
