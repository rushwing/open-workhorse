# Scenario

当 Bug 开立触发 REQ blocking（`blocked_reason=bug_linked`）时，`blocked_from_owner` 须被写入；
当 Bug 关闭触发 unblock 时，REQ `owner` 须从 `blocked_from_owner` 恢复，且字段随后被清空。

# Steps

1. 准备一个 `status=req_review, owner=huahua` 的测试 REQ stub
2. 调用 block 逻辑（模拟 bug-standard.md §2.2）：
   - 写入 `blocked_reason: bug_linked`
   - 写入 `blocked_from_status: req_review`
   - 写入 `blocked_from_owner: huahua`
   - 将 `owner` 改为 `unassigned`
   - 将 `status` 改为 `blocked`
3. 运行 `check-req-coverage.sh`：期望 exit 0（`blocked_from_owner` 校验通过）
4. 调用 unblock 逻辑（模拟 bug-standard.md §2.3）：
   - 将 `status` 恢复为 `blocked_from_status`（req_review）
   - 将 `owner` 恢复为 `blocked_from_owner`（huahua）
   - 清空 `blocked_reason`、`blocked_from_status`、`blocked_from_owner`
5. 验证最终 REQ：`status=req_review, owner=huahua, blocked_from_owner=""`

# Expected

- Step 3：`check-req-coverage.sh` exit 0，无关于 `blocked_from_owner` 的报错
- Step 5：`owner` 已从 `unassigned` 恢复为 `huahua`（非 `unassigned`）
- Step 5：`blocked_from_owner` 字段已清空（`""` 或空）
- negative：若 step 2 跳过写入 `blocked_from_owner`，则 `check-req-coverage.sh` exit 1，输出含 "blocked_from_owner 为空或缺失"
