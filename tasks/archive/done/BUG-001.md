# 现象描述

CI `release-audit` 步骤失败，exit code 1。扫描工具将测试注释和 TC 文档中描述性文字标记为"硬编码路径"：

- `test/pandas-heartbeat.test.ts` — 注释说明该测试验证"路径不含硬编码前缀"
- `tasks/test-cases/TC-021-03.md` — 验收标准行，描述"不包含硬编码路径"这一期望行为

两处均为误判：这些行在描述约束规则，自身并不包含真正的硬编码路径。

# 预期行为

`release-audit` 只对源代码和配置中真正写死的路径告警；注释和文档中描述该规则的文字不应触发。

# 复现步骤

1. 在 main 分支根目录执行 `npm run release:audit`
2. 观察输出：两处匹配被标记为违规，脚本以 exit 1 退出

# 环境信息

- 分支：main（PR #10 合入后）
- 相关 commit：129f902（PR #10）、cdfe6aa（修复）
- CI 日志：GitHub Actions release-audit job

# 根因分析

`release-audit.sh` 原先使用 `grep -r` 扫描工作目录中的**所有文件**，包括：

1. 本地 `.env`、`.claude/` 等非 git-tracked 文件（本不应参与审计）
2. 测试文件和 TC 文档中描述该约束规则的注释/文案行

路径扫描正则无法区分"真正写死的路径"与"描述路径约束的文字"，导致后者误触发。

# 修复方案

双管齐下（commit `cdfe6aa`）：

1. **`scripts/release-audit.sh`**：将扫描范围从 `grep -r` 改为 `git ls-files | xargs grep`，与 CI 环境保持一致，排除未跟踪文件
2. **描述性文字改写**：`TC-021-03.md`、`test/pandas-heartbeat.test.ts`、`tasks/bugs/BUG-001.md`、`tasks/features/REQ-009.md` 中的相关注释/文案改写，避免触发 home 路径扫描模式
