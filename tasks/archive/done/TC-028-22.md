# Scenario

用户在 GitHub 提交新 issue 后，Pandas 同步检测到该 issue 尚未有 `bug-tracked` label，开立对应本地 `BUG-xxx.md` 并在 GitHub issue 上打 `bug-tracked` label。

per bug-standard.md §8.1：Pandas 分配 BUG 编号，创建文件，填写必要字段，在 GitHub issue 上打 label `bug-tracked`，commit message 格式为 `bug: create BUG-xxx from GitHub issue #<n>`。

# Steps

1. Mock GitHub API：`gh.issueList()` 返回 issue `{number: 99, title: "Button crash on submit", labels: []}` （无 `bug-tracked` label）
2. Mock `gh.issueAddLabel` 函数；mock 文件系统写入（或使用临时目录）
3. 运行 Pandas 开立 user_bug 流程
4. 断言创建了 `tasks/bugs/BUG-xxx.md`，文件内容含：
   - `bug_type: user_bug`
   - `github_issue: 99`
   - `status: open`
5. 断言 `gh.issueAddLabel(99, "bug-tracked")` 被调用
6. 断言 commit message 匹配 `bug: create BUG-xxx from GitHub issue #99`
7. 构造第二个场景：issue #99 已有 `bug-tracked` label
8. 再次运行流程
9. 断言 `gh.issueAddLabel` 在第二次运行中**未被再次调用**（幂等）
10. 断言未创建重复的 BUG 文件

# Expected

- 首次检测到无 bug-tracked label 的 issue：创建本地 BUG-xxx.md（含必要字段），打 bug-tracked label
- 已有 bug-tracked label 的 issue：跳过创建，不重复操作（幂等）
- 本地 BUG 文件 status 初始为 open，bug_type 为 user_bug
