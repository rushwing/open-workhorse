# Scenario

impl_bug 从 open 推进到 closed 的完整主路径。关联 REQ 从 in_progress → blocked → 恢复 in_progress（不跳到 review）。

# Steps

1. 构造 bug fixture：`bug_type=impl_bug, status=open, related_req=[REQ-001]`；构造 REQ fixture：`status=in_progress`
2. 调用 transition(open → confirmed)；断言 `bug.status=confirmed`
3. 断言 `REQ-001.status=blocked, REQ-001.blocked_reason=bug_linked, REQ-001.blocked_from_status=in_progress`
4. 调用 transition(confirmed → in_progress)；断言 `bug.owner=menglan`（impl_bug fix 责任人为 menglan）
5. 调用 transition(in_progress → fixed)；断言 `bug.status=fixed`
6. 调用 transition(fixed → regressing)；断言 `bug.status=regressing`
7. 调用 transition(regressing → closed)；断言 `bug.status=closed`
8. 断言 `REQ-001.status=in_progress`（恢复 blocked_from_status，不应跳至 review）
9. 断言 `REQ-001.blocked_reason` 字段为空或不存在

# Expected

- fix 责任人=menglan（符合 impl_bug 路由规则）
- REQ unblock 回到 in_progress，而非跳到 req_review 或其他更高状态
- blocked_reason 和 blocked_from_status 字段清空
