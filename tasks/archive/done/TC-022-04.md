# Scenario

`.harness_sessions` 中有一条超过阈值时间的 `implement REQ-XXX` 记录，
对应任务仍为 `in_progress`，`stall_detection` 应调用 `tg_notify`。

# Steps

1. 创建 `.harness_sessions` 包含：`2020-01-01T00:00:00Z implement REQ-999`
2. 创建 `tasks/features/REQ-999.md`（status=in_progress, owner=claude_code）
3. Mock `tg_notify` 为打印函数（避免真实 Telegram 调用）
4. 调用 `stall_detection`
5. 断言 mock tg_notify 被调用

# Expected

- `stall_detection` 检测到 REQ-999 停滞并调用 `tg_notify`
- tg_notify 消息包含 "REQ-999" 和 "停滞"
