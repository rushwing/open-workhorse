# Scenario

同一 REQ 的第一条消息已进入 done/，再次发出 request。

# Steps

1. 调用 `inbox_write "menglan" "implement" "REQ-035" "first"` 生成第一条消息
2. 将该消息从 `for-menglan/pending/` 移到 `for-menglan/done/`（模拟已处理）
3. 再次调用 `inbox_write "menglan" "implement" "REQ-035" "second"`

# Expected

- 两条消息的 `thread_id` 相同
- 两条消息的 `correlation_id` 不同
