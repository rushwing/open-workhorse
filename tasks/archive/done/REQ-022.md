# Goal

将 `scripts/pandas-heartbeat.sh` 接入 OpenClaw heartbeat 机制，使 Pandas 编排循环随 OpenClaw 心跳自动推进。

设计细节见 `docs/PANDAS-ORCHESTRATION.md` §4。

# In Scope

- `src/index.ts` 中新增 `APP_COMMAND=pandas-heartbeat` 分发分支，调用 `scripts/pandas-heartbeat.sh`
- `scripts/pandas-heartbeat.sh` 实现三项职责（按顺序）：
  1. 处理 `inbox/for-pandas/` 中的所有消息（调用 REQ-021 的 `inbox_read_pandas`）
  2. Auto-claim：扫描 `tasks/features/` 找 status=ready/test_designed + owner=unassigned，选最高优先级认领
  3. 停滞检测：复用 `dev-cycle-watchdog.sh` 逻辑，超阈值则 Telegram 告警

# Out of Scope

- heartbeat 调度频率（由 OpenClaw 控制，非本 REQ 范围）
- Telegram 发送实现（由 REQ-024 提供）
- 消息路由到 Huahua/Menglan（由 REQ-023 实现）

# Acceptance Criteria

- `APP_COMMAND=pandas-heartbeat node --env-file-if-exists=.env src/index.ts` 执行后调用 `scripts/pandas-heartbeat.sh`
- 脚本在 inbox 为空时正常退出（exit 0）
- Auto-claim 正确按优先级选取最高优先级未认领任务
- `bash -n scripts/pandas-heartbeat.sh` 无语法错误
- `npm run build` 通过（src/index.ts 变更）
- `npm run release:audit` 通过
- `npm run req:check` 通过

# Test Case Design Notes

- TC-022-01: APP_COMMAND=pandas-heartbeat 触发脚本（mock execFile 或集成测试）
- TC-022-02: inbox 为空时，脚本正常退出，无错误输出
- TC-022-03: tasks/ 有 P2 和 P1 未认领任务时，auto-claim 选择 P1
- TC-022-04: 活跃任务超过 DEV_WATCHDOG_STALE_HOURS 时，脚本调用 Telegram 告警
