# Scenario

user_bug 到达 regressing 后，GitHub issue 被用户关闭，同步后本地 status 更新为 closed。本地不调用 gh.issueClose（因为用户已关闭）。

# Steps

1. 构造 bug fixture：`bug_type=user_bug, status=regressing, github_issue=42`
2. Mock `gh.issueView(42)` 返回 `{state: "CLOSED"}`
3. 运行 GitHub 同步流程
4. 断言 `bug.status=closed`
5. 断言 `bug.agent_notes` 含关闭来源说明（如 "closed by GitHub user"）
6. 断言 `gh.issueClose` **未被调用**（issue 已由用户关闭，无需再次调用）

# Expected

- 同步检测到 GitHub issue 已关闭，本地 status 更新为 closed
- Agent Notes 记录关闭来源
- 不重复调用 gh.issueClose
