## Summary

两个关联改进，减少 HITL 手动操作并提升链路健壮性。

## Sub-A：单 PR 规则

Hua Hua TC 设计和 Meng Lan 需求开发共用同一个 `feat/REQ-N` 分支和同一个 PR。
Daniel 每个需求只需 merge 一次（归档 PR 除外）。

### Acceptance Criteria

1. Hua Hua `req_review` 完成后，在 `feat/${req_id}` 分支（而非 `tc/...`）创建 TC 文件并开 PR。
2. `tc_review` 消息携带 `branch_name` 字段（值为 `feat/${req_id}`）。
3. 该字段沿消息链传递：tc_review → tc_complete → implement。
4. Meng Lan 收到 `implement` 消息时，若携带 `branch_name`，以 `EXISTING_BRANCH` 环境变量传给 `harness.sh implement`。
5. `harness.sh cmd_worktree_setup`：若远端已有 `feat/${req_id}` 分支，fetch+checkout，不重新建分支。
6. `harness.sh cmd_implement` prompt：若 `EXISTING_BRANCH` 非空，检查已有 PR 并更新描述（而非新建 PR）。
7. `tc_policy=exempt/optional` 路径行为不变（无 branch_name，正常新建分支和 PR）。

## Sub-B：Keep-Alive Watchdog

Pandas 5 分钟心跳扫描 Menglan 的 `in_progress` 任务，若对应存活时间戳超过 60 分钟（可配置）则发送去重的 keep-alive 消息触发恢复。

### Acceptance Criteria

1. `menglan-heartbeat.sh` 每次运行在 `runtime/menglan_alive.ts` 写当前 epoch 时间戳。
2. `huahua-heartbeat.sh` 每次运行在 `runtime/huahua_alive.ts` 写当前 epoch 时间戳。
3. `pandas-heartbeat.sh` 新增 `_check_stall_and_keepalive()`：
   - 扫描 `tasks/features/REQ-*.md`（status=in_progress，owner=menglan）
   - 若 `runtime/menglan_alive.ts` 不存在或时间戳距今 > `AGENT_STALL_TIMEOUT_MINUTES`（默认 60）分钟：
     - 向 Menglan inbox 写去重后的 keep-alive implement 请求
     - `tc_policy` 非 `optional/exempt` 时带上 `branch_name: feat/${req_id}`
4. `.env.example` 新增 `# AGENT_STALL_TIMEOUT_MINUTES=60` 配置项。
