# Scenario

`harness.sh implement REQ-037` を 2 回連続で呼び出したとき、2 回目は worktree の
重複作成エラーを起こさず exit 0 で終了すること（幂等性保証）。

# Steps

1. 1 回目：`./scripts/harness.sh implement REQ-037` を実行 → worktree 作成
2. `git worktree list` で worktree エントリ数を記録
3. 2 回目：`./scripts/harness.sh implement REQ-037` を再実行
4. `git worktree list` で worktree エントリ数が増加していないことを確認
5. スクリプトの終了コードを確認

# Expected

- 2 回目の実行が exit 0 で終了すること
- `git worktree list` のエントリ数が 1 回目と同じであること（重複エントリなし）
- ログに「worktree 已存在」または相当するメッセージが出力されること
