# Scenario

tc_bug 从 open 推进到 closed 的完整主路径。关联 REQ 从 ready → blocked → 恢复 ready，最终目标状态为 test_designed。

# Steps

1. 构造 bug fixture：`bug_type=tc_bug, status=open, related_req=[REQ-001]`；构造 REQ fixture：`status=ready`
2. 调用 transition(open → confirmed)；断言 `bug.status=confirmed, bug.owner=pandas`
3. 断言 `REQ-001.status=blocked, REQ-001.blocked_reason=bug_linked, REQ-001.blocked_from_status=ready`
4. 调用 transition(confirmed → in_progress)；断言 `bug.owner=huahua`（tc_bug fix 责任人为 huahua）
5. 断言 `bug.reviewer=menglan`（tc_bug reviewer 为 menglan）
6. 调用 transition(in_progress → fixed)；断言 `bug.status=fixed`
7. 调用 transition(fixed → regressing)；断言 `bug.status=regressing`
8. 调用 transition(regressing → closed)；断言 `bug.status=closed`
9. 断言 `REQ-001.status=ready`（恢复 blocked_from_status）
10. 断言 `REQ-001.blocked_reason` 字段为空或不存在

# Expected

- fix 责任人=huahua，reviewer=menglan（符合 tc_bug 路由规则）
- REQ unblock 后恢复至 ready（blocked_from_status），而非 test_designed 或其他状态
- blocked_reason 和 blocked_from_status 字段清空
