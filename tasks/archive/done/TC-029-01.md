# Scenario

`check-req-coverage.sh` 的 STATUS_ENUM 新增 `review_ready` 后，含该状态的 REQ 文件应通过校验。

# Steps

1. 在 `tasks/features/` 创建临时 REQ 文件（如 `REQ-TEST-029-01.md`），frontmatter 含 `status: review_ready`，其他必填字段合法（req_id、title、priority、phase 等）
2. 执行 `bash scripts/check-req-coverage.sh`
3. 检查退出码和输出

# Expected

- 退出码：0
- 输出中该 REQ 显示通过标记（✓ 或无错误行）
- 不出现 "值 'review_ready' 不在允许枚举" 或类似拒绝信息
