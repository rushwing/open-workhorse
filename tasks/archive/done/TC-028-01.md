# Scenario

req_bug 从 open 推进到 closed 的完整主路径。触发时关联 REQ 从 req_review → blocked(blocked_reason=bug_linked)；关闭时 REQ 恢复 req_review，blocked_reason 清空。

# Steps

1. 构造 bug fixture：`bug_type=req_bug, status=open, related_req=[REQ-001]`；构造 REQ fixture：`status=req_review`
2. 调用 transition(open → confirmed)；断言 `bug.status=confirmed, bug.owner=pandas`
3. 断言 `REQ-001.status=blocked, REQ-001.blocked_reason=bug_linked, REQ-001.blocked_from_status=req_review`
4. 调用 transition(confirmed → in_progress)；断言 `bug.owner=menglan`
5. 调用 transition(in_progress → fixed)；断言 `bug.status=fixed`
6. 调用 transition(fixed → regressing)；断言 `bug.status=regressing`
7. 调用 transition(regressing → closed)；断言 `bug.status=closed`
8. 断言 `REQ-001.status=req_review`（恢复 blocked_from_status）
9. 断言 `REQ-001.blocked_reason` 字段为空或不存在
10. 断言 `REQ-001.blocked_from_status` 字段为空或不存在

# Expected

- 每步 status/owner 符合 bug-standard.md §2.4 路由表
- REQ unblock 后恢复至 `blocked_from_status`（req_review），而非跳至其他状态
- blocked_reason 和 blocked_from_status 字段清空
