# 现象描述

REQ-042 frontmatter 中 `scope: src,tests`，其中 `src` 不在 `check-req-coverage.sh` 允许枚举 `{runtime, ui, tests, scripts, docs, harness}` 内，且字段使用逗号分隔多值格式，与单值规范不符。

# 预期行为

`scope` 字段应为单个合法枚举值之一：`runtime`、`ui`、`tests`、`scripts`、`docs` 或 `harness`。

REQ-042 涉及 `src/contracts/` 和 `src/lib/` 下的 TypeScript 源文件，正确值应为 `runtime`（源码层）。

# 根因分析

Daniel 编写 REQ-042 时使用了 `scope: src,tests`（逗号分隔多值），不符合 requirement-standard.md §5.1 的单值枚举约束。`src` 不在允许枚举内，正确对应值为 `runtime`。

# 修复方案

将 `scope: src,tests` 修改为 `scope: runtime`。测试覆盖在 Agent Notes 中说明。

# 回归测试

`bash scripts/check-req-coverage.sh` → REQ-042 基础检查通过 ✓
