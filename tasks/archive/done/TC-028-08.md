# Scenario

Reviewer 连续打回 3 次（fixed → in_progress），第 3 次触发 blocked，blocked_reason=review_round_exceeded，并发送 TG 通知。

# Steps

1. 构造 bug fixture：`status=in_progress, owner=menglan, review_round=0`；mock tg_notify 函数
2. 第 1 次：transition(in_progress → fixed) → reviewer 打回 → transition(fixed → in_progress)；断言 `review_round=1`，`status=in_progress`
3. 第 2 次：transition(in_progress → fixed) → reviewer 打回 → transition(fixed → in_progress)；断言 `review_round=2`，`status=in_progress`
4. 第 3 次：transition(in_progress → fixed) → reviewer 打回 → transition(fixed → in_progress 或 blocked)
5. 断言 `review_round=3`
6. 断言 `bug.status=blocked`
7. 断言 `bug.blocked_reason=review_round_exceeded`
8. 断言 `tg_notify` mock 被调用至少 1 次（通知 Daniel 介入）

# Expected

- review_round 每次打回递增
- review_round=3 时自动转 blocked，blocked_reason=review_round_exceeded
- tg_notify 被调用（通知人工介入，不依赖 Agent 自行处理）
