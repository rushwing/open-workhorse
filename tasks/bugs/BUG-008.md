# 现象描述

REQ-042.md 在 feat/REQ-042 分支上（TC 设计提交后）存在两个 frontmatter 字段错误：

1. `scope: runtime,tests` — scope 字段使用了多值，违反 requirement-standard.md §5.1 的枚举约束（必须为 `runtime | ui | tests | scripts | docs` 单值之一）
2. `acceptance` frontmatter 摘要写的是 `inboxRead()`，而规范文档 In Scope 节、Acceptance Criteria 第 3 条和 TC-042-09 一致使用 `inboxReadLegacy()`，造成函数名不一致

# 预期行为

- `scope: runtime`（单值，与 main 分支 REQ-042.md 一致，与 BUG-007 已修复的方向相同）
- `acceptance` frontmatter 摘要使用 `inboxReadLegacy()` 而非 `inboxRead()`

1. `git checkout feat/REQ-042`

3. 查看 `acceptance` 第三行：`inboxWrite()、inboxRead()、inboxClaim()...`

# 根因分析

> TC 设计提交（b00f6bd）在更新 REQ-042.md frontmatter 时引入了两处错误：
> 1. scope 字段值被改为 `runtime,tests`（多值），与 main 上已修正的 `runtime` 不一致，且 BUG-007 刚因同类问题关闭
> 2. acceptance 摘要中函数名拼写为 `inboxRead()` 而非规范要求的 `inboxReadLegacy()`

# 修复方案

> 在 feat/REQ-042 分支上修改 `tasks/features/REQ-042.md`：
> 1. 将 `scope: runtime,tests` 改为 `scope: runtime`
> 2. 将 `acceptance` 摘要中的 `inboxRead()` 改为 `inboxReadLegacy()`
