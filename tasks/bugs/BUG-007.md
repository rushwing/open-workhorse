# 现象描述

REQ-042 frontmatter 中 `scope: src,tests`，其中 `src` 不在 `check-req-coverage.sh` 允许枚举 `{runtime, ui, tests, scripts, docs, harness}` 内，且字段使用逗号分隔多值格式，与单值规范不符。

# 预期行为

`scope` 字段应为单个合法枚举值之一：`runtime`、`ui`、`tests`、`scripts`、`docs` 或 `harness`。

REQ-042 涉及 `src/contracts/` 和 `src/lib/` 下的 TypeScript 源文件，正确值应为 `runtime`（源码层）或按需拆分为独立 scope——但不能使用 `src` 或逗号列表。

# 复现步骤

1. `bash scripts/check-req-coverage.sh`
2. 观察输出：`REQ-042.scope: 值 'src' 不在允许枚举 [runtime ui tests scripts docs harness] 内`

# 环境信息

- 分支：main
- 相关文件：tasks/features/REQ-042.md

# 根因分析

> 待修复者填写

# 修复方案

将 `scope: src,tests` 修改为合法的单值，例如 `scope: runtime`（该 REQ 主要产出物为 `src/contracts/` 和 `src/lib/` 下的 TypeScript 模块，属于运行时/库层）。如需同时标注测试文件，建议在 Agent Notes 中说明，scope 字段保持单值。
