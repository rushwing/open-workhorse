# 现象描述

CI `release-audit` 步骤失败，exit code 1。扫描工具将以下两处描述性文字标记为"硬编码 macOS/Linux home 路径"（机器特定绝对主目录前缀）：

- `test/pandas-heartbeat.test.ts:121` — 一条注释，说明该测试验证路径不含硬编码前缀
- `tasks/test-cases/TC-021-03.md:25` — 验收标准行，描述"不包含硬编码路径"这一期望行为

两处均为误判：这些行在描述约束规则，自身并不包含真正的硬编码路径。

# 预期行为

`release-audit` 只对源代码和配置中真正写死的路径告警；注释和文档中描述该规则的文字不应触发。

# 复现步骤

1. 在 main 分支根目录执行 `npm run release:audit`
2. 观察输出：两处匹配被标记为违规，脚本以 exit 1 退出
