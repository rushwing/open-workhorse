需求流转规程 v0.4

requirement-standard.md v0.4 · Open Workhorse · 9 states · Pandas 只锚定入口/出口，中间路径 Huahua 自持

泳道图 — 3 Agent 协作路径

① Pandas 心跳只触发两次：review_ready→req_review（入口）和 review→done（出口）。中间所有 handoff 由 Huahua/Menglan 直写 inbox，无等待。
② 折叠路径（虚线）：Huahua 在 req_review 阶段同步完成 TC 设计，直接跳到 test_designed，跳过 ready。
③ impl_bug 打回（红色）：code review 失败 → impl_bug → REQ blocked → Menglan fix → re-PR → 右侧弧线回到 code review。
④ req_bug 打回：req_review 评审失败 → Huahua 开立 req_bug → Menglan 修复需求文档 → pending_bugs:[] → unblock → req_review。
⑤ tc_bug 打回：tc_review 评审失败 → Menglan 开立 tc_bug → Huahua 修复 TC 文件 → pending_bugs:[] → unblock → test_designed。
⑥ ci_bug：done 后 Pandas 监测 CI 失败 → Menglan 开立 ci_bug 修复 → CI green → Bug closed（REQ 保持 done）。

v0.4 关键字段

字段 / 状态类型说明
review_ready新状态（第 9 个）Human-to-Pipeline 交接点。Daniel 设置；Pandas 原子 commit 转为 req_review（state-as-lock 防重复认领）
pending_bugs新 frontmatter 字段BUG-xxx 数组；非空时 REQ 进入 blocked；Bug 关闭后从此字段移除并追加到 REQ ## 关联 Bug 历史
blocked_from_status已有字段进入 blocked 前的状态，unblock 时恢复用
